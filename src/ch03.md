# 第 3 章 · 字符串、数组与对象

## 本章目标

前两章我们学了变量、类型、运算符、控制流和函数基础，手里有了"零件"。但真实程序处理的数据很少是单个数字或单个字符——而是**一段文字**、**一批数据**、**一个结构化的实体**。本章就来认识 JavaScript 里三种最重要的数据载体：

- **字符串（string）**：处理文本。掌握常用方法，理解"字符串不可变（immutable）"。
- **数组（array）**：处理一批有序数据。重点是 `map`/`filter`/`reduce` 等**函数式方法**——这是本章的重中之重，也是现代 JS 的核心表达方式。
- **对象（object）**：处理带命名字段的结构化数据。掌握读写、遍历、合并。
- **解构赋值（destructuring）**：一种优雅地从数组/对象里"取值"的语法。
- **引用类型 vs 值类型**：搞懂 `=` 到底赋了什么，避开浅拷贝、对象比较的坑。
- **JSON**：对象与文本之间的桥梁（本章先简介，第 6 章深入）。

学完本章，你就能写出处理真实数据的程序了。章末的小项目——**待办清单数据模型**——会把这些知识拧成一股绳。

## 3.1 字符串常用方法

字符串是一串字符。虽然它看起来"简单"，但 JavaScript 给它配了一整套方法。我们先看一个集中演示，再逐一拆解。

```javascript
// 文件：code/ch03/01-string.js
const s = "Hello, JavaScript";

// 1) length：字符个数（不是方法，没有括号）
console.log("长度：", s.length); // 17

// 2) 索引：像数组一样用方括号取单个字符，从 0 开始
console.log("第 0 个字符：", s[0]); // "H"
console.log("最后一个字符：", s[s.length - 1]); // "t"
```

- **`length`** 是**属性**不是方法，所以没有括号。它数的是"字符个数"。
- **索引访问** `s[0]` 取第一个字符。⚠️ 记住 JS 里下标从 **0** 开始，所以最后一个字符的下标是 `length - 1`，不是 `length`。`s[17]` 会得到 `undefined`。

### 截取子串：slice 与 substring

```javascript
// 文件：code/ch03/01-string.js
console.log("slice(0, 5)：", s.slice(0, 5)); // "Hello"
console.log("slice(7)：", s.slice(7)); // "JavaScript"
console.log("slice(-6)：", s.slice(-6)); // "Script"

console.log("substring(0, 5)：", s.substring(0, 5)); // "Hello"
console.log("substring(-6)：", s.substring(-6)); // 负数当 0，等于整串
```

- **`slice(start, end)`** 截取从 `start` 到 `end` 的部分，但**不包含 `end`**——这是一个"左闭右开"区间 `[start, end)`。整个标准库到处是这种约定，务必记牢。
- 只给一个参数 `slice(7)` 表示"从 7 一直到末尾"。
- `slice` 支持**负数**：`-6` 表示"倒数第 6 个字符开始"，`slice(-6)` 就取到了 `"Script"`。
- **`substring`** 和 `slice` 很像，但它**不支持负数**（把负数当 0 处理）。💡 记不住就统一用 `slice`，功能更全。

### 查找：indexOf 与 includes

```javascript
// 文件：code/ch03/01-string.js
console.log("indexOf('Java')：", s.indexOf("Java")); // 7
console.log("indexOf('Python')：", s.indexOf("Python")); // -1（找不到）
console.log("includes('Script')：", s.includes("Script")); // true
```

- **`indexOf(子串)`** 返回子串第一次出现的**下标**；⚠️ 找不到时返回 **`-1`**（不是 `undefined`，也不是报错）。所以判断"是否包含"的老写法是 `s.indexOf("x") !== -1`。
- **`includes(子串)`** 更直白，直接返回 `true`/`false`，可读性更好，现在优先用它。

### 拆分与拼接：split 与 join

```javascript
// 文件：code/ch03/01-string.js
const csv = "苹果,香蕉,橙子";
const fruits = csv.split(","); // ["苹果","香蕉","橙子"]
console.log("split：", fruits);
console.log("join：", fruits.join(" | ")); // "苹果 | 香蕉 | 橙子"
```

- **`split(分隔符)`** 把字符串按分隔符切成一个**数组**。这是把文本变成可处理数据的常用第一步（比如解析 CSV）。
- **`join(连接符)`** 是数组的方法，作用相反：把数组每一项用连接符粘成一个字符串。`split` 和 `join` 是一对好搭档。

### 替换：replace 与 replaceAll

```javascript
// 文件：code/ch03/01-string.js
console.log("replace：", "a-b-c".replace("-", "+")); // 只替换第一个："a+b-c"
console.log("replaceAll：", "a-b-c".replaceAll("-", "+")); // 全部："a+b+c"
```

⚠️ 新手最容易踩的坑：**`replace` 默认只替换第一个匹配**！想全部替换，用 **`replaceAll`**（或配合正则表达式，见第 12 章）。

### 大小写、去空白、补齐、重复

```javascript
// 文件：code/ch03/01-string.js
console.log("toUpperCase：", s.toUpperCase()); // "HELLO, JAVASCRIPT"
console.log("toLowerCase：", s.toLowerCase()); // "hello, javascript"

console.log("trim：", "   有空格   ".trim() + "<--"); // "有空格<--"

console.log("padStart：", "5".padStart(3, "0")); // "005"
console.log("padEnd：", "5".padEnd(3, "0")); // "500"

console.log("repeat：", "=".repeat(10)); // "=========="
```

- **`toUpperCase` / `toLowerCase`**：统一大小写，常用于"忽略大小写的比较"（先都转小写再比）。
- **`trim`**：去掉**首尾**空白（中间的不动）。处理用户输入时几乎必用——谁知道用户会不会手滑多打空格。
- **`padStart(目标长度, 填充字符)`**：把字符串补到指定长度，不够就在**前面**填。`"5".padStart(3, "0")` → `"005"`，常用于对齐数字、补零。`padEnd` 则填在后面。
- **`repeat(n)`**：重复 n 次，常用来画分隔线。

### 模板字符串复习

```javascript
// 文件：code/ch03/01-string.js
const name = "小明";
const age = 18;
console.log(`我叫 ${name}，今年 ${age} 岁，明年 ${age + 1} 岁。`);
```

第 1 章讲过：用反引号 `` ` `` 包起来、内部用 `${表达式}` 嵌入变量或表达式，就是**模板字符串（template literal）**。`${}` 里可以放任何表达式（如 `age + 1`）。它比用 `+` 拼接清爽太多，本书后面会大量使用。

### 📌 字符串不可变（immutable）

这是一个**极其重要**的概念：字符串一旦创建，**内容就不能改变**。所有"看起来在改字符串"的方法，其实都是**返回一个新字符串**，原字符串纹丝不动。

```javascript
// 文件：code/ch03/01-string.js
const original = "hello";
const upper = original.toUpperCase();
console.log("原串没变：", original); // "hello"
console.log("新串：", upper); // "HELLO"

// 试图用索引修改字符是无效的：普通模式下静默失败，严格模式/模块里会报错。
// 想"改字符"，本质是用方法生成一个新字符串，再赋回去。
const fixed = "H" + original.slice(1);
console.log("要改就生成新串：", fixed, "原串仍是：", original); // "Hello" "hello"
```

拆解：

1. `original.toUpperCase()` 没有改 `original`，而是**返回了新串** `"HELLO"`，赋给了 `upper`。`original` 依然是 `"hello"`。
2. 你可能想 `original[0] = "H"` 直接改第一个字符——**这行不通**。在普通脚本里它静默失败（假装没发生），在严格模式或 ES 模块里干脆报错。字符串的字符是只读的。
3. 想要一个"改过的字符串"，正确姿势是**用方法拼出一个新串**再赋值。

> 📌 记住这句话：**字符串方法从不修改原串，只返回新串。** 想保留结果，一定要用变量接住返回值。这个特性和下面要讲的数组形成鲜明对比——数组的很多方法会"就地修改"。

## 3.2 数组基础

数组（array）是一批**有序**数据的容器。它用方括号 `[]` 创建，元素之间用逗号隔开，可以放任何类型（甚至混着放，但实践中通常放同类数据）。

```javascript
// 文件：code/ch03/02-array-basics.js
const nums = [10, 20, 30];
console.log("第 0 个：", nums[0]); // 10
console.log("长度：", nums.length); // 3
console.log("越界访问：", nums[99]); // undefined（不报错）

nums[0] = 100;
nums[3] = 40; // 直接给新索引赋值也能扩展
console.log("修改后：", nums); // [100, 20, 30, 40]
```

- 数组也用**下标**访问，从 0 开始，`nums.length` 是元素个数。
- 越界访问 `nums[99]` 返回 `undefined`，不会报错——这点要小心，越界不报错反而容易埋 bug。
- ⚠️ 注意：虽然数组用 `const` 声明，我们却能改它的元素、给它加元素！`const` 只保证"变量不能重新指向另一个数组"，但**数组内容是可变（mutable）的**。这和字符串完全不同。

### 末尾/开头增删：push / pop / shift / unshift

```javascript
// 文件：code/ch03/02-array-basics.js
const stack = [1, 2];
const newLen = stack.push(3); // 末尾加 3，返回新长度 3
console.log("push 后：", stack, "新长度：", newLen); // [1,2,3] 3
const popped = stack.pop(); // 弹出末尾元素 3
console.log("pop 后：", stack, "弹出：", popped); // [1,2] 3

const queue = [1, 2];
queue.unshift(0); // 开头加 0
console.log("unshift 后：", queue); // [0,1,2]
const first = queue.shift(); // 移除开头
console.log("shift 后：", queue, "移除：", first); // [1,2] 0
```

记忆口诀（都是**就地修改原数组**）：

| 方法 | 位置 | 动作 | 返回值 |
| --- | --- | --- | --- |
| `push` | 末尾 | 加 | 新长度 |
| `pop` | 末尾 | 删 | 被删的元素 |
| `unshift` | 开头 | 加 | 新长度 |
| `shift` | 开头 | 删 | 被删的元素 |

💡 push/pop 操作末尾很快；unshift/shift 操作开头需要移动所有后续元素，数据量大时较慢。

### 万能增删改：splice

```javascript
// 文件：code/ch03/02-array-basics.js
const letters = ["a", "b", "c", "d"];
const removed = letters.splice(1, 2, "X", "Y", "Z"); // 从索引1删2个，插入3个
console.log("splice 后：", letters); // ["a","X","Y","Z","d"]
console.log("被删除的：", removed); // ["b","c"]
```

**`splice(start, deleteCount, ...要插入的元素)`** 是数组增删改的"瑞士军刀"：从 `start` 位置删掉 `deleteCount` 个，再把后面的元素插进去。它**就地修改原数组**，返回被删掉的元素组成的数组。

⚠️ 别把 `splice`（改原数组）和下面的 `slice`（不改原数组）搞混，名字太像了。

### 截取与拼接：slice / concat

```javascript
// 文件：code/ch03/02-array-basics.js
const arr = [0, 1, 2, 3, 4];
console.log("slice(1, 3)：", arr.slice(1, 3)); // [1,2]
console.log("slice(-2)：", arr.slice(-2)); // [3,4]
console.log("原数组没变：", arr); // [0,1,2,3,4]

const a = [1, 2];
const b = [3, 4];
console.log("concat：", a.concat(b, [5])); // [1,2,3,4,5]
console.log("a 没变：", a); // [1,2]
```

- 数组的 **`slice(start, end)`** 和字符串的一样：左闭右开、支持负数、**返回新数组、不改原数组**。
- **`concat`** 把多个数组拼成一个新数组，也不改原数组。

### 扩展运算符 `...`（重要）

```javascript
// 文件：code/ch03/02-array-basics.js
console.log("展开合并：", [...a, ...b, 5]); // [1,2,3,4,5]
const copy = [...a]; // 快速浅拷贝一个数组
copy.push(99);
console.log("拷贝改动不影响原数组：", a, copy); // [1,2] [1,2,99]
```

**扩展运算符（spread）`...`** 把一个数组"摊开"，倒进另一个方括号里。它有两个超高频用法：

1. **合并数组**：`[...a, ...b, 5]` 比 `concat` 更直观。
2. **浅拷贝数组**：`[...a]` 得到一个新数组，内容相同但是独立的。后面讲"引用类型"你就懂这为什么重要了。

💡 现代 JS 里，`...` 几乎取代了 `concat` 和很多手写循环，非常值得用熟。

## 3.3 数组的函数式方法（本章核心）

这一节是全书前几章里最重要的内容之一。现代 JavaScript 处理数组，很少再写 `for` 循环一个个搬，而是用一组"传入一个函数、自动帮你遍历"的方法。这种风格叫**函数式（functional）**，代码更短、更清晰、更少出错。

所有这些方法都接收一个**回调函数（callback）**——你告诉它"对每个元素做什么"，它负责循环。回调常见签名是 `(元素, 索引, 整个数组) => ...`，大多数时候我们只用前一两个参数。

我们用一个示例文件贯穿讲解：

```javascript
// 文件：code/ch03/03-array-methods.js
const nums = [1, 2, 3, 4, 5, 6];
```

### forEach：只遍历，不返回

```javascript
// 文件：code/ch03/03-array-methods.js
nums.forEach((n, i) => {
  console.log(`索引 ${i} 的值是 ${n}`);
});
```

**`forEach`** 相当于"漂亮版 for 循环"：对每个元素执行回调。它**没有返回值**（返回 `undefined`），所以它只适合"做点副作用"（比如打印、累加到外部变量），不适合"生成新数据"。要生成新数据，用下面的 `map`。

### map：一对一映射

```javascript
// 文件：code/ch03/03-array-methods.js
const doubled = nums.map((n) => n * 2);
console.log("翻倍：", doubled); // [2,4,6,8,10,12]
console.log("原数组没变：", nums); // [1,2,3,4,5,6]
```

**`map`** 是最常用的方法之一：它对每个元素调用回调，把**回调的返回值**收集成一个**等长的新数组**。上面"每个数翻倍"，得到长度相同的新数组，原数组不变。

📌 记住 `map` 的本质：**输入 n 个 → 输出 n 个，一对一变形**。想从 `[{name:"小明"}, ...]` 里"抽出所有 name"，就是 `.map(u => u.name)`——这是日常最常见的用途。

### filter：按条件筛选

```javascript
// 文件：code/ch03/03-array-methods.js
const evens = nums.filter((n) => n % 2 === 0);
console.log("偶数：", evens); // [2,4,6]
```

**`filter`** 的回调返回一个"要不要保留"的布尔值：返回 `true` 就保留，`false` 就丢弃。结果是一个**可能更短**的新数组。上面留下了所有偶数。

📌 `map` 改变每个元素的"值"，`filter` 改变数组的"长度"（筛掉一些）。这两个是黄金搭档。

### reduce：归并成一个值

`reduce` 是这几个方法里最抽象、也最强大的一个。它把整个数组"归并（reduce）"成**一个**结果——可能是一个数字、一个字符串，甚至一个新对象。

```javascript
// 文件：code/ch03/03-array-methods.js
const sum = nums.reduce((acc, n) => acc + n, 0);
console.log("求和：", sum); // 21
```

`reduce(回调, 初始值)` 的回调签名是 `(累加器 acc, 当前元素 n) => 新的累加器`。它的工作过程是：

1. `acc` 一开始等于**初始值** `0`。
2. 遍历到第一个元素 `1`：`acc = 0 + 1 = 1`。
3. 遍历到 `2`：`acc = 1 + 2 = 3`。
4. …一路累加到 `21`，作为最终结果返回。

把 `acc` 想象成一个"随身携带、不断更新的账本"，每遍历一个元素就更新一次。理解了求和，其他用法就好懂了：

```javascript
// 文件：code/ch03/03-array-methods.js
const max = nums.reduce((acc, n) => (n > acc ? n : acc), nums[0]);
console.log("最大值：", max); // 6

const words = ["a", "b", "a", "c", "b", "a"];
const count = words.reduce((acc, w) => {
  acc[w] = (acc[w] || 0) + 1;
  return acc;
}, {});
console.log("词频统计：", count); // { a: 3, b: 2, c: 1 }
```

- **求最大值**：账本 `acc` 存"目前见过的最大值"，每次和当前元素比较，留大的。初始值用第一个元素。
- **词频统计**：这次账本是一个**对象** `{}`！每见到一个词，就把它对应的计数 `+1`（`acc[w] || 0` 是"没有就当 0"的常用技巧）。⚠️ 注意回调里一定要 `return acc`，否则下一轮账本就变成 `undefined` 了——这是新手用 reduce 最常见的错误。

> 💡 什么时候用 reduce？当你想把一个数组"塌缩"成单个值（求和、求最值、拼字符串、按某字段分组统计成一个对象……）时。如果只是"一对一变形"用 `map`，"筛选"用 `filter`，别硬套 reduce。

### find / findIndex：找第一个

```javascript
// 文件：code/ch03/03-array-methods.js
console.log("第一个 > 3 的：", nums.find((n) => n > 3)); // 4
console.log("它的索引：", nums.findIndex((n) => n > 3)); // 3
console.log("找不到时 find：", nums.find((n) => n > 99)); // undefined
console.log("找不到时 findIndex：", nums.findIndex((n) => n > 99)); // -1
```

- **`find`** 返回**第一个**满足条件的**元素**；找不到返回 `undefined`。
- **`findIndex`** 返回第一个满足条件的元素的**下标**；找不到返回 `-1`。

它们和 `filter` 的区别：`filter` 找**所有**符合的（返回数组），`find` 只要**第一个**（返回单个）。按 id 找一条记录时，`find` 正是你要的。

### some / every：整体判断

```javascript
// 文件：code/ch03/03-array-methods.js
console.log("有没有偶数：", nums.some((n) => n % 2 === 0)); // true
console.log("是不是都 > 0：", nums.every((n) => n > 0)); // true
```

- **`some`**：只要**有一个**元素满足条件就返回 `true`（"存在"）。
- **`every`**：**所有**元素都满足才返回 `true`（"全部"）。

它们返回布尔值，常用于校验，比如"表单里是否有一项没填"用 `some`，"是否所有项都合法"用 `every`。

### sort：排序（含著名陷阱）

```javascript
// 文件：code/ch03/03-array-methods.js
const bad = [10, 2, 1, 20, 3].sort(); // 默认按字符串，结果反直觉
console.log("默认 sort（陷阱）：", bad); // [1, 10, 2, 20, 3]

const good = [10, 2, 1, 20, 3].sort((a, b) => a - b); // 传比较函数才对
console.log("升序：", good); // [1, 2, 3, 10, 20]

const desc = [10, 2, 1, 20, 3].sort((a, b) => b - a);
console.log("降序：", desc); // [20, 10, 3, 2, 1]
```

⚠️ **这是 JS 最有名的坑之一**：`sort()` 不传参数时，会把每个元素**转成字符串再按字典序比较**！所以 `10` 排在了 `2` 前面（因为字符 `"1"` < `"2"`）。这几乎肯定不是你想要的。

正确做法是**传一个比较函数** `(a, b) => ...`，规则是：

- 返回**负数** → `a` 排在 `b` 前面；
- 返回**正数** → `a` 排在 `b` 后面；
- 返回 `0` → 两者顺序不变。

所以数字升序用 `a - b`（小的在前），降序用 `b - a`。

还有一个陷阱：**`sort` 会就地修改原数组**！如果你不想破坏原数组，先拷贝再排：

```javascript
// 文件：code/ch03/03-array-methods.js
const src = [3, 1, 2];
const sorted = [...src].sort((a, b) => a - b);
console.log("原数组保护：", src, "排序结果：", sorted); // [3,1,2] [1,2,3]
```

### flat / flatMap：展平嵌套

```javascript
// 文件：code/ch03/03-array-methods.js
console.log("flat：", [1, [2, 3], [4, [5]]].flat()); // [1,2,3,4,[5]]
console.log("flat(2)：", [1, [2, [3, [4]]]].flat(2)); // [1,2,3,[4]]

const sentences = ["hello world", "foo bar"];
console.log("flatMap：", sentences.flatMap((s) => s.split(" ")));
// ["hello","world","foo","bar"]
```

- **`flat`** 把嵌套数组"压平"一层；传个数字 `flat(2)` 压平两层。
- **`flatMap`** = 先 `map` 再 `flat` 一层，专治"一个元素变成多个"的场景。上面把每个句子拆成词，再合并成一个大词表。

### 链式调用：函数式的威力

这些方法大多返回新数组，于是可以**串起来**，像流水线一样一步步加工数据：

```javascript
// 文件：code/ch03/03-array-methods.js
const result = nums
  .filter((n) => n % 2 === 0) // 先筛偶数 [2,4,6]
  .map((n) => n * 10) // 再各乘 10 [20,40,60]
  .reduce((acc, n) => acc + n, 0); // 再求和 120
console.log("偶数各乘10再求和：", result); // 120
```

读这段代码几乎像读一句话："筛出偶数，各乘 10，然后求和。" 这种**声明式**的表达，正是函数式方法最迷人的地方。本章的小项目会大量用到这种链式写法。

## 3.4 对象基础

数组用**下标**（位置）访问数据；而对象（object）用**名字**（键 key）访问数据。当一个东西有多个"带名字的字段"时——比如一个用户有 name、age、city——就用对象来表示。

```javascript
// 文件：code/ch03/04-object.js
const user = {
  name: "小明",
  age: 18,
  isStudent: true,
};

console.log("点语法：", user.name); // "小明"
console.log("方括号：", user["age"]); // 18
const key = "isStudent";
console.log("动态 key 只能用方括号：", user[key]); // true
```

- 对象用**花括号 `{}`** 创建，里面是一组 `键: 值` 对，这叫**对象字面量（object literal）**。键（也叫**属性 property**）是名字，值可以是任何类型。
- 读属性有两种语法：**点语法** `user.name`（最常用）和**方括号语法** `user["name"]`。
- 什么时候必须用方括号？当**属性名存在变量里**（`user[key]`）、或属性名含特殊字符/空格时。点语法要求你把属性名"写死"在代码里。

### 增、删、改属性

```javascript
// 文件：code/ch03/04-object.js
user.age = 19; // 修改
user.city = "北京"; // 新增（对象是可变的）
delete user.isStudent; // 删除
console.log("改动后：", user); // { name: "小明", age: 19, city: "北京" }
```

- 给已有属性赋值就是**改**；给不存在的属性赋值就是**新增**。
- 用 **`delete`** 关键字删除属性。
- 和数组一样，`const` 声明的对象**内容照样可改**——`const` 只是不让你把 `user` 指向另一个对象。

### 嵌套与方法

```javascript
// 文件：code/ch03/04-object.js
const order = {
  id: 1001,
  customer: { name: "小红", vip: true },
  items: ["书", "笔"],
};
console.log("嵌套读取：", order.customer.name); // "小红"
console.log("对象里的数组：", order.items[0]); // "书"

const counter = {
  count: 0,
  increment() {
    this.count += 1; // this 指向 counter
    return this.count;
  },
};
console.log("方法调用：", counter.increment()); // 1
console.log("再调用：", counter.increment()); // 2
```

- 对象的值可以是另一个对象或数组，层层嵌套，用点/方括号一路取下去。真实数据几乎都是这种嵌套结构。
- 值是函数的属性叫**方法（method）**。`increment()` 里的 **`this`** 指向"调用这个方法时点前面的那个对象"，这里是 `counter`。`this` 是个大话题，第 4 章会专门讲，现在只需知道"方法里的 `this` 通常指向自己所属的对象"。

### 简写属性与计算属性名

```javascript
// 文件：code/ch03/04-object.js
const title = "待办";
const done = false;
const todo = { title, done }; // 等价于 { title: title, done: done }
console.log("简写属性：", todo); // { title: "待办", done: false }

const field = "score";
const record = {
  name: "小明",
  [field]: 95, // 属性名是 field 的值 "score"
  [`${field}_max`]: 100,
};
console.log("计算属性名：", record); // { name:"小明", score:95, score_max:100 }
```

- **简写属性（shorthand）**：当变量名和想要的属性名相同时，`{ title }` 就是 `{ title: title }` 的简写。这个语法在实际代码里随处可见。
- **计算属性名（computed property name）**：把属性名用**方括号**括起来，里面放表达式，用它的**值**当属性名。`[field]: 95` 生成的属性名是 `"score"`。当属性名要动态决定时非常有用。

### 遍历对象：Object.keys / values / entries

对象不能直接用数组那套 `map`/`filter`。要遍历对象，先用这三个方法把它"摊成数组"：

```javascript
// 文件：code/ch03/04-object.js
const p = { a: 1, b: 2, c: 3 };
console.log("keys：", Object.keys(p)); // ["a","b","c"]
console.log("values：", Object.values(p)); // [1,2,3]
console.log("entries：", Object.entries(p)); // [["a",1],["b",2],["c",3]]

for (const [k, v] of Object.entries(p)) {
  console.log(`  ${k} = ${v}`);
}
```

- **`Object.keys(obj)`**：所有键组成的数组。
- **`Object.values(obj)`**：所有值组成的数组。
- **`Object.entries(obj)`**：所有 `[键, 值]` 对组成的数组。

`Object.entries` 配合 `for...of` 和**数组解构** `[k, v]`（下一节讲）来遍历键值对，是极常用的组合。拿到 keys/values 数组后，你就能对它们用 `map`/`filter`/`reduce` 了——对象和数组由此打通。

### 合并对象：Object.assign 与展开

```javascript
// 文件：code/ch03/04-object.js
const defaults = { theme: "light", fontSize: 14 };
const custom = { fontSize: 16 };
const merged1 = Object.assign({}, defaults, custom); // 后面覆盖前面
const merged2 = { ...defaults, ...custom }; // 更常用的写法
console.log("assign 合并：", merged1); // { theme:"light", fontSize:16 }
console.log("展开合并：", merged2); // { theme:"light", fontSize:16 }

const updated = { ...defaults, theme: "dark" };
console.log("生成新对象：", updated, "原对象没变：", defaults);
```

- **`Object.assign(目标, ...来源)`** 把后面对象的属性拷进目标对象；**后面的覆盖前面的**（所以 `fontSize` 变 16）。传空对象 `{}` 当目标，可以避免改到 `defaults`。
- **对象展开 `{ ...obj }`** 是更现代、更常用的写法，语义等价：把 `obj` 的属性"摊"进新对象。
- 一个高频模式：`{ ...defaults, theme: "dark" }` 表示"基于旧对象，只改一个字段，生成一个新对象"，原对象保持不变。⚠️ 这在后面的小项目和第 26 章的状态管理里会反复出现——记住它。

## 3.5 解构赋值

解构赋值（destructuring）是一种"从数组/对象里批量取值"的简洁语法。它不引入新能力，但能让代码短很多、清晰很多，现代 JS 代码里到处都是。

### 数组解构

```javascript
// 文件：code/ch03/05-destructuring.js
const point = [10, 20, 30];
const [x, y, z] = point;
console.log("数组解构：", x, y, z); // 10 20 30

const [first, , third] = point; // 逗号占位跳过中间
console.log("跳过中间：", first, third); // 10 30

const [head, ...tail] = [1, 2, 3, 4]; // ...rest 收集剩余
console.log("head：", head, "tail：", tail); // 1 [2,3,4]

let a = 1;
let b = 2;
[a, b] = [b, a]; // 一行交换两个变量
console.log("交换后：", a, b); // 2 1
```

- **数组解构**按**位置**取值：`[x, y, z] = [10, 20, 30]` 一次性把三个值分别赋给三个变量。
- 用**逗号占位**可以跳过不想要的元素。
- `...tail` 是**剩余（rest）语法**，把"剩下的元素"收成一个数组。⚠️ 注意区分：出现在"取值/参数"这边是 rest（收集），出现在"赋值/展开"那边是 spread（摊开），符号都是 `...`。
- `[a, b] = [b, a]` 是不用临时变量交换两个值的经典技巧。

### 对象解构

```javascript
// 文件：code/ch03/05-destructuring.js
const user = { name: "小明", age: 18, city: "北京" };
const { name, age } = user;
console.log("对象解构：", name, age); // 小明 18

const { name: userName } = user; // 重命名
console.log("重命名：", userName); // 小明

const { gender = "未知" } = user; // 默认值（属性不存在时）
console.log("默认值：", gender); // 未知

const { role: userRole = "普通用户" } = user; // 重命名 + 默认值
console.log("重命名+默认值：", userRole); // 普通用户
```

- **对象解构**按**名字**取值：`{ name, age } = user` 从 `user` 里取出同名属性。注意这里不看位置，只看键名。
- **重命名** `{ name: userName }`：取出 `name`，但用 `userName` 这个变量名接住。（别和"设置属性"搞混，解构里冒号左边是源属性名，右边是新变量名。）
- **默认值** `{ gender = "未知" }`：当属性不存在（`undefined`）时用默认值。
- 两者可以叠加：`{ role: userRole = "普通用户" }`。

### 嵌套解构与函数参数解构

```javascript
// 文件：code/ch03/05-destructuring.js
const order = {
  id: 1001,
  customer: { name: "小红", address: { city: "上海" } },
};
const {
  customer: {
    name: customerName,
    address: { city },
  },
} = order;
console.log("嵌套解构：", customerName, city); // 小红 上海
```

嵌套结构可以照着"形状"一层层解构下去，直接取到深层的值。

解构最实用的场景是**函数参数**。传一个"配置对象"、在参数位置直接解构出需要的字段并给默认值，是现代 JS 库最常见的 API 风格：

```javascript
// 文件：code/ch03/05-destructuring.js
function createUser({ name, age = 0, vip = false } = {}) {
  return `${name}, ${age}岁, ${vip ? "VIP" : "普通"}`;
}
console.log(createUser({ name: "小李", age: 25, vip: true }));
console.log(createUser({ name: "小王" })); // age、vip 用默认值
console.log(createUser()); // 连参数都没传，靠末尾的 = {} 兜底
```

- 参数写成 `{ name, age = 0, vip = false }`：调用者传一个对象，函数内自动解构出这几个字段，缺的用默认值。好处是**调用时不用记参数顺序**，只需按名字给。
- 末尾的 `= {}` 很关键：如果调用者**一个参数都不传**，就用空对象兜底，否则对 `undefined` 解构会报错。💡 这是"带默认配置的函数"的标准模板。

## 3.6 引用类型 vs 值类型

这一节回答一个看似简单、却坑了无数新手的问题：**`=` 到底赋了什么？**

JavaScript 的类型分两类：

- **值类型（原始类型 primitive）**：number、string、boolean、null、undefined、symbol、bigint。赋值时复制"值本身"。
- **引用类型（reference）**：object、array、function（本质都是对象）。赋值时复制的是"引用"（可以理解成对象的内存地址）。

```javascript
// 文件：code/ch03/06-reference.js
let n1 = 10;
let n2 = n1; // 复制了一份值
n2 = 20;
console.log("值类型：", n1, n2); // 10 20（互不影响）

const obj1 = { count: 1 };
const obj2 = obj1; // obj2 和 obj1 指向同一个对象
obj2.count = 999;
console.log("引用类型：", obj1.count, obj2.count); // 999 999（一起变了！）
console.log("是同一个对象吗：", obj1 === obj2); // true
```

- 值类型：`n2 = n1` 复制了数值 `10`，之后两者独立。改 `n2` 不影响 `n1`。
- 引用类型：`obj2 = obj1` **没有复制对象**，只是让 `obj2` 和 `obj1` 指向**同一个对象**。所以通过 `obj2` 改 `count`，`obj1` 看到的也变了——因为它俩本就是同一个东西。数组同理。

📌 一句话记住：**对象/数组的 `=` 赋的是"引用"，不是"副本"。想要独立的副本，必须显式拷贝。**

### 浅拷贝 vs 深拷贝

```javascript
// 文件：code/ch03/06-reference.js
const source = { name: "小明", hobbies: ["篮球"] };
const shallow = { ...source }; // 浅拷贝（或 Object.assign({}, source)）
shallow.name = "小红"; // 第一层独立，改了不影响原对象
console.log("浅拷贝改第一层：", source.name, shallow.name); // 小明 小红
shallow.hobbies.push("足球"); // ⚠️ 内层数组仍是共享的引用！
console.log("浅拷贝改内层：", source.hobbies); // ["篮球","足球"] 也被改了！
```

**浅拷贝（shallow copy）**——`{...source}` 或 `Object.assign`——只复制**第一层**。第一层的原始值（如 `name`）是独立的，改了互不影响；但第一层里如果是个对象/数组（如 `hobbies`），复制的仍是那个引用，**内层还是共享的**！这是极其常见的隐蔽 bug。

要连内层一起复制，用**深拷贝（deep copy）**。现代 Node/浏览器内置了 `structuredClone`：

```javascript
// 文件：code/ch03/06-reference.js
const source2 = { name: "小明", hobbies: ["篮球"] };
const deep = structuredClone(source2); // 深拷贝
deep.hobbies.push("足球");
console.log("深拷贝改内层：", source2.hobbies, deep.hobbies);
// ["篮球"] ["篮球","足球"]  —— 互不影响
```

💡 需要真正独立的完整副本时，`structuredClone(obj)` 是目前最方便的选择。此外也有人用 `JSON.parse(JSON.stringify(obj))` 做深拷贝，但它有局限（函数、`undefined`、`Date` 会丢失或变形），下一节会看到。

### 对象比较的坑

```javascript
// 文件：code/ch03/06-reference.js
const p1 = { x: 1 };
const p2 = { x: 1 };
console.log("内容相同但不是同一个：", p1 === p2); // false！
console.log("和自己比：", p1 === p1); // true
console.log("按内容比：", JSON.stringify(p1) === JSON.stringify(p2)); // true
```

⚠️ **`===` 比较对象时，比的是"是不是同一个对象（同一个引用）"，而不是内容！** 所以两个内容一模一样的对象 `p1 === p2` 也是 `false`。想按内容比较，简易办法是转成 JSON 字符串再比（有局限，复杂场景见第 6 章）。

### 函数传参也是传引用

```javascript
// 文件：code/ch03/06-reference.js
function addTag(o) {
  o.tag = "已处理"; // 直接改会影响外部对象
}
const data = { id: 1 };
addTag(data);
console.log("函数改了外部对象：", data); // { id:1, tag:"已处理" }
```

把对象传给函数，函数拿到的是**同一个对象的引用**。函数内部改它的属性，外面也会变。这既是特性也是陷阱：如果你不想让函数改到外部数据，就在函数里先拷贝一份再操作——本章小项目的所有函数都遵循"不改传入的旧数据、返回新数据"这一原则，正是为了避开这类问题。

## 3.7 JSON 简介

**JSON**（JavaScript Object Notation）是一种**文本格式**，长得几乎和 JS 对象字面量一样，用来在"程序内的对象"和"可存储/可传输的文本"之间转换。存文件、发网络请求，几乎都用 JSON。本章先认个脸，第 6 章会深入。

```javascript
// 文件：code/ch03/07-json.js
const user = {
  name: "小明",
  age: 18,
  hobbies: ["篮球", "编程"],
  address: { city: "北京" },
};

const json = JSON.stringify(user);
console.log("stringify：", json);
console.log("类型：", typeof json); // string

console.log("美化输出：");
console.log(JSON.stringify(user, null, 2)); // 第三参数 2 = 缩进 2 空格
```

- **`JSON.stringify(对象)`** 把对象/数组转成 **JSON 字符串**（注意结果类型是 `string`）。
- 加上参数 `JSON.stringify(user, null, 2)`，会输出带缩进的漂亮格式，便于人阅读或写进配置文件。

反过来：

```javascript
// 文件：code/ch03/07-json.js
const text = '{"id":1,"title":"买牛奶","done":false}';
const obj = JSON.parse(text);
console.log("parse：", obj);
console.log("访问字段：", obj.title); // 买牛奶
```

**`JSON.parse(字符串)`** 把 JSON 字符串还原成 JS 对象，之后就能正常访问字段了。从文件读进来的、从网络接口拿到的，通常都是字符串，第一步就是 `JSON.parse`。

⚠️ 用 JSON 做深拷贝虽方便，但**有损**——函数、`undefined` 会消失，`Date` 会变成字符串：

```javascript
// 文件：code/ch03/07-json.js
const tricky = { fn: () => 1, u: undefined, when: new Date(0) };
console.log("有损转换：", JSON.stringify(tricky));
// {"when":"1970-01-01T00:00:00.000Z"}  —— fn 和 u 都没了
```

所以需要可靠深拷贝时，优先用上一节的 `structuredClone`。

## 常见坑与最佳实践

把本章反复强调的"陷阱"集中列一遍，遇到 bug 时可以回来对照：

1. ⚠️ **字符串不可变**：所有字符串方法都返回新串、不改原串。忘了用变量接住返回值，你的"修改"就白做了。
2. ⚠️ **`replace` 只替换第一个**：要全部替换用 `replaceAll`。
3. ⚠️ **`sort()` 默认按字符串排序**：排数字一定要传比较函数 `(a, b) => a - b`。而且 `sort` 会改原数组，需要保护原数组就先 `[...arr]` 拷贝。
4. ⚠️ **`slice` 不改原数组，`splice` 改原数组**：名字像，行为反，别用混。
5. ⚠️ **`reduce` 的回调忘了 `return`**：账本会变成 `undefined`，结果全错。
6. ⚠️ **对象/数组是引用**：`=` 不拷贝内容；`{...obj}`/`[...arr]` 只是浅拷贝，内层仍共享；要完整独立副本用 `structuredClone`。
7. ⚠️ **`===` 比对象比的是引用不是内容**：两个内容相同的对象不相等。
8. 💡 **优先用不可变风格**：需要"改"数据时，用 `map`/`filter`/展开生成**新**数组/对象，而不是就地 `push`/`splice`/改属性。数据流更可预测，也更容易调试——这正是下面小项目采用的风格。
9. 💡 **`map`/`filter`/`reduce` 各司其职**：变形用 `map`，筛选用 `filter`，塌缩成一个值用 `reduce`。别把什么都硬塞进 `reduce`。

## 本章小项目：待办清单数据模型

现在把本章知识拧成一股绳，做一个**待办清单（Todo List）的数据模型**。注意：这里**只做数据层**——用数组 + 对象表示一批待办，并实现一组操作它们的**纯函数**（添加、删除、切换完成、筛选、统计、排序、格式化）。交互界面留到后面章节（第 4 章会复用这些函数，第 23 章会给它接上 REST API）。

**核心设计原则：所有函数都是"纯函数"**——不修改传入的旧数组，而是**返回一个新数组**。这样数据流清晰可预测，也为将来做"撤销/重做""状态管理"打好基础。这正是 3.6 节强调的不可变风格的实战。

一条待办事项的"形状"约定如下：

```text
{
  id: number,        // 唯一标识
  title: string,     // 标题
  done: boolean,     // 是否完成
  createdAt: number, // 创建时间（时间戳，毫秒）
}
```

### 完整实现

```javascript
// 文件：code/ch03/todo-model.js

// ---------- 1) 生成 id ----------
// 简单起见，用"当前最大 id + 1"。真实项目里会用 uuid（第 24 章）。
function nextId(todos) {
  const maxId = todos.reduce((max, t) => (t.id > max ? t.id : max), 0);
  return maxId + 1;
}

// ---------- 2) 添加 ----------
function addTodo(todos, title) {
  const newTodo = {
    id: nextId(todos),
    title,
    done: false,
    createdAt: Date.now(),
  };
  return [...todos, newTodo]; // 用展开运算符拼出新数组
}

// ---------- 3) 删除 ----------
function removeTodo(todos, id) {
  return todos.filter((t) => t.id !== id);
}

// ---------- 4) 标记完成 / 切换状态 ----------
function toggleTodo(todos, id) {
  return todos.map((t) => (t.id === id ? { ...t, done: !t.done } : t));
}

// ---------- 5) 按状态筛选 ----------
// status: "all" | "active"(未完成) | "completed"(已完成)
function filterTodos(todos, status) {
  switch (status) {
    case "active":
      return todos.filter((t) => !t.done);
    case "completed":
      return todos.filter((t) => t.done);
    default:
      return todos; // "all"
  }
}

// ---------- 6) 统计未完成数量 ----------
function countActive(todos) {
  return todos.reduce((n, t) => (t.done ? n : n + 1), 0);
}

// ---------- 7) 按创建时间排序 ----------
function sortByCreatedAt(todos, order = "asc") {
  const sign = order === "desc" ? -1 : 1;
  return [...todos].sort((a, b) => (a.createdAt - b.createdAt) * sign);
}

// ---------- 8) 格式化打印成好看的清单 ----------
function formatTodos(todos) {
  if (todos.length === 0) return "（清单为空）";

  const lines = todos.map((t) => {
    const box = t.done ? "[x]" : "[ ]"; // 完成打勾
    const idText = `#${t.id}`.padEnd(4, " "); // 对齐 id
    return `${box} ${idText} ${t.title}`;
  });

  const active = countActive(todos);
  const total = todos.length;
  const footer = `—— 共 ${total} 项，未完成 ${active} 项 ——`;

  return [...lines, footer].join("\n");
}

export {
  nextId,
  addTodo,
  removeTodo,
  toggleTodo,
  filterTodos,
  countActive,
  sortByCreatedAt,
  formatTodos,
};
```

### 逐段讲解

1. **`nextId`（reduce 求最大值）**：用 `reduce` 遍历现有待办，账本 `max` 存"目前最大的 id"，返回 `+1`。空数组时初始值 `0`，于是第一条 id 是 1。

2. **`addTodo`（展开生成新数组）**：构造一个新待办对象（`title` 用了**简写属性**，`done` 初始为 `false`，`createdAt` 用 `Date.now()` 取当前毫秒时间戳），然后用 `[...todos, newTodo]` **拼出一个新数组**返回——原数组完全没动。这就是不可变风格。

3. **`removeTodo`（filter 删除）**：用 `filter` 留下"id 不等于目标"的所有项，被删的自然就被排除了。同样返回新数组。

4. **`toggleTodo`（map + 展开改单项）**：用 `map` 遍历，只有 id 命中的那一项，用 `{ ...t, done: !t.done }` 生成一个"其余字段照抄、只翻转 `done`"的**新对象**；其余项原样返回。这是 3.4 节"基于旧对象改一个字段生成新对象"模式的实战。

5. **`filterTodos`（filter + switch）**：按 `"active"`/`"completed"`/`"all"` 三种状态返回不同筛选结果，`switch` 的 `default` 兜底返回全部。

6. **`countActive`（reduce 计数）**：用 `reduce` 数出未完成（`!done`）的数量。也可以写成 `todos.filter(t => !t.done).length`，两种都对；这里用 reduce 是为了再练一次它。

7. **`sortByCreatedAt`（拷贝后 sort）**：先 `[...todos]` 拷贝再 `sort`，避免改原数组（3.3 节的陷阱）。比较函数用 `createdAt` 相减，`sign` 控制升序/降序。

8. **`formatTodos`（map + join 排版）**：用 `map` 把每条待办渲染成一行文本（完成的画 `[x]`，用 `padEnd` 对齐 id），最后拼上一行统计，用 `join("\n")` 连成多行字符串。这里 `map`、`padEnd`、模板字符串、`join` 全用上了。

9. **`export`**：把这些纯函数导出，方便后续章节复用（本书 `code/` 目录采用 ES 模块，所以用 `export`；模块化细节见第 9 章）。

### 演示各操作（使用这个模型）

下面是同一个文件里"使用"这些函数的部分：

```javascript
// 文件：code/ch03/todo-model.js（演示部分）
let todos = [];
todos = addTodo(todos, "学习字符串方法");
todos = addTodo(todos, "学习数组方法");
todos = addTodo(todos, "完成本章小项目");
todos = addTodo(todos, "预习第 4 章");

console.log("【添加 4 项后】");
console.log(formatTodos(todos));

todos = toggleTodo(todos, 1);
todos = toggleTodo(todos, 2);
console.log("【标记 #1、#2 完成后】");
console.log(formatTodos(todos));

todos = removeTodo(todos, 4);
console.log("【删除 #4 后】");
console.log(formatTodos(todos));

console.log("【只看未完成 active】");
console.log(formatTodos(filterTodos(todos, "active")));

console.log("【统计】未完成数量：", countActive(todos));

console.log("【按创建时间降序】");
console.log(formatTodos(sortByCreatedAt(todos, "desc")));
```

注意每次操作我们都写 `todos = 某函数(todos, ...)`——因为函数不改原数组、只返回新数组，我们用返回值**覆盖**变量来"推进状态"。这种"旧状态 → 新状态"的更新方式，正是现代前端状态管理（React、Redux 等）的核心思想，你在这里已经提前体会到了。
