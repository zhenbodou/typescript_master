上一章我们把环境装好、跑通了第一个程序。从这一章开始，我们正式进入 JavaScript 语言本身。**变量、数据类型、运算符**是所有编程语言的三块基石——你写的每一行代码，几乎都在"用运算符处理某种类型的数据，并把结果存进变量"。把这一章吃透，后面所有内容都会顺很多。

## 1.1 什么是变量

想象你在厨房做菜，需要几个**贴了标签的盒子**来临时装东西：一个盒子贴"糖"、一个贴"盐"。做菜时你不用记"糖在第三个柜子左边"，只要说"拿糖那个盒子"就行。

**变量（variable）就是这样一个贴了名字的盒子，用来存放数据。** 你给它起个名字，往里放值，之后用名字就能取出、也能换成别的值。

在 JavaScript 里，声明变量要先用一个**关键字**：`let` 或 `const`（还有个老旧的 `var`，稍后说为什么别用）。

```javascript
// 文件：code/ch01/01-variables.js
// 运行方式：node 01-variables.js

// 用 let 声明一个"可以改变"的变量。
let score = 10; // 声明并赋初值
console.log("初始分数：", score);

score = 25; // 重新赋值，完全合法
console.log("加分后：", score);

// 用 const 声明一个"常量"：一旦赋值，就不能再指向别的值。
const pi = 3.14159;
console.log("圆周率：", pi);

// 下面这行如果取消注释，运行会直接报错：
// TypeError: Assignment to constant variable.
// pi = 3.14;

// 命名要见名知意：好的名字本身就是注释。
let userAge = 18; // 驼峰命名（camelCase），推荐
const MAX_RETRY = 3; // 全大写 + 下划线，常表示"配置型常量"
console.log("用户年龄：", userAge, "，最大重试：", MAX_RETRY);
```

我们逐段拆解：

- **`let score = 10;`** —— 这行做了两件事：**声明**（告诉 JS "我要一个叫 `score` 的盒子"）和**赋值**（`=` 号把右边的 `10` 装进去）。注意 `=` 在编程里读作"赋值"，意思是"把右边的值放进左边的盒子"，**不是数学里的"相等"**。这一点初学者极易搞混，请务必记牢。

- **`score = 25;`** —— 因为用的是 `let`，这个盒子里的东西**可以换**。这里不再写 `let`（变量已经声明过了，再写 `let score` 会报"重复声明"），直接换值即可。

- **`const pi = 3.14159;`** —— `const` 是 constant（常量）的缩写，表示"声明后就不许再指向别的值"。如果你取消那行注释去改 `pi`，程序会直接崩溃报 `TypeError`。

- **命名**：`userAge` 用的是**驼峰命名法（camelCase）**——第一个单词小写，后面每个单词首字母大写，像驼峰一样起伏。这是 JS 社区的主流约定。`MAX_RETRY` 用全大写加下划线，习惯上用来表示"写死的配置常量"。

📌 **`let` 还是 `const`，怎么选？** 一条简单可靠的规则：**默认全用 `const`，只有当你确实需要之后改这个变量时，才改用 `let`。** 这样做的好处是，别人（和未来的你）看到 `const` 就知道"这个值不会变"，读代码时脑子里要记的东西更少，也能让引擎提前拦住"不小心改错"的 bug。

⚠️ **命名规则（硬性）**：变量名可以由字母、数字、下划线 `_`、美元符 `$` 组成，但**不能以数字开头**（`2cat` 非法，`cat2` 合法）；不能用 JS 的保留字（如 `let`、`if`、`return`）当名字；区分大小写（`age` 和 `Age` 是两个不同的变量）。

### 为什么不推荐 `var`

在 2015 年（ES6）之前，声明变量只有一个关键字 `var`。今天你会在老代码里见到它，但**新代码请一律用 `let` / `const`**，原因主要有两个：

1. **`var` 没有"块级作用域"**。`let`/`const` 声明的变量只在它所在的 `{ }` 花括号内有效；而 `var` 会"泄漏"到整个函数里，容易造成意料之外的相互覆盖。
2. **`var` 允许重复声明同一个名字而不报错**，掩盖了本该暴露的错误。

作用域的细节我们留到第 4 章展开。现在你只要记住结论：**忘掉 `var`，用 `const` 和 `let`。**

## 1.2 基本数据类型

盒子里能装什么样的"东西"？JavaScript 把数据分成几种**类型（type）**。其中最基础的一批叫**原始类型（primitive）**，一共 7 种。它们的特点是"值本身"很简单、不可再分（相对第 3 章的对象、数组这些"复合类型"而言）。

我们先把它们全部见一遍：

```javascript
// 文件：code/ch01/02-types.js
// 运行方式：node 02-types.js

// ===== 1. number（数字）=====
const intNum = 42; // 整数
const floatNum = 3.14; // 小数
const negNum = -7; // 负数
console.log("number：", intNum, floatNum, negNum);

// number 的三个坑：
console.log("0.1 + 0.2 =", 0.1 + 0.2); // 0.30000000000000004（浮点精度）
console.log("1 / 0 =", 1 / 0); // Infinity（无穷大）
console.log("0 / 0 =", 0 / 0); // NaN（Not a Number，不是一个数字）
console.log("NaN === NaN ?", NaN === NaN); // false！NaN 谁都不等，包括它自己

// ===== 2. string（字符串）=====
const single = 'Hello'; // 单引号
const double = "World"; // 双引号，二者等价
console.log("string：", single, double);

// ===== 3. boolean（布尔）=====
const isDone = true;
const isEmpty = false;
console.log("boolean：", isDone, isEmpty);

// ===== 4. undefined 和 null =====
let notAssigned; // 只声明、不赋值 -> undefined
const nothing = null; // 主动表示"这里就是空"
console.log("undefined：", notAssigned);
console.log("null：", nothing);

// ===== 5. bigint（大整数，简介）=====
const big = 9007199254740993n; // 结尾加 n 表示 bigint
console.log("bigint：", big);

// ===== 6. symbol（唯一标识，简介）=====
const id = Symbol("id"); // 每个 symbol 都独一无二
console.log("symbol：", id.toString());

// ===== 用 typeof 查看类型 =====
console.log("--- typeof 一览 ---");
console.log(typeof 42); // "number"
console.log(typeof "hi"); // "string"
console.log(typeof true); // "boolean"
console.log(typeof undefined); // "undefined"
console.log(typeof 10n); // "bigint"
console.log(typeof Symbol()); // "symbol"
console.log(typeof null); // "object"（历史遗留 bug，见正文）
console.log(typeof console.log); // "function"
```

下面逐个类型讲。

### number：数字

JavaScript **不区分整数和小数**，统统叫 `number`。`42`、`3.14`、`-7` 都是同一种类型。这和很多语言（有 `int`、`float`、`double` 之分）不同，对初学者反而更省心。

但 number 有三个必须知道的坑：

**坑一：浮点精度。** 运行会看到 `0.1 + 0.2` 竟然等于 `0.30000000000000004`，而不是 `0.3`。这**不是 JavaScript 的 bug**，几乎所有语言都这样。原因是计算机用二进制存小数，而 `0.1` 这种数在二进制里是"无限循环小数"，只能存一个近似值，多个近似值相加就产生了微小误差。**结论：永远不要直接用 `===` 判断两个小数是否相等。** 需要时先四舍五入（如 `(0.1 + 0.2).toFixed(2)` 得到 `"0.30"`），或比较它们的差是否足够小。

**坑二：`Infinity`（无穷大）。** `1 / 0` 在 JS 里不报错，而是返回 `Infinity`。还有 `-Infinity`。它们是合法的 number 值。

**坑三：`NaN`。** `NaN` 是 "Not a Number"（不是一个数字）的缩写，表示"一次本该得到数字、却失败了的运算"，比如 `0 / 0`。它有个反直觉到极点的性质：**`NaN` 不等于任何值，包括它自己**，所以 `NaN === NaN` 是 `false`。想判断一个值是不是 `NaN`，不能用 `=== NaN`，要用专门的函数 `Number.isNaN(x)`（本章小项目里会用到）。

### string：字符串

**字符串（string）就是一段文本**，用引号包起来。可以用单引号 `'...'` 或双引号 `"..."`，两者完全等价，选一种风格保持一致即可。字符串我们下一节（1.3）专门细讲，这里先混个眼熟。

### boolean：布尔

**布尔（boolean）只有两个值：`true`（真）和 `false`（假）**，用来表达"是/否""开/关""成立/不成立"。它是所有判断逻辑（第 2 章的 `if`）的基础。

### undefined 和 null：两种"空"

这两个都表示"没有值"，但含义不同，初学者常混淆：

- **`undefined`** 是"系统给的空"：你声明了变量却没赋值（`let notAssigned;`），JS 自动把它设成 `undefined`，意思是"这盒子还空着，还没人放东西"。
- **`null`** 是"你主动给的空"：`const nothing = null;`，表示"我明确知道这里没有值、就是空"。

📌 一个记忆口诀：**`undefined` 是"还没赋值"，`null` 是"故意置空"。** 大多数时候，让 JS 用 `undefined` 表示"缺失"就好；当你想主动表达"这里清空了"时才用 `null`。

### bigint 和 symbol（简介）

这两个用得少，简单认识即可：

- **`bigint`** 用来表示**超大整数**。普通 number 能安全表示的最大整数约是 9 千万亿（`2^53 - 1`），超过就会丢精度。在数字末尾加个 `n`（如 `9007199254740993n`）就得到 bigint，可以精确表示任意大的整数。日常算钱、算年龄用不到，处理超大 ID 或高精度计算时才需要。
- **`symbol`** 用来生成**独一无二的标识**。`Symbol("id")` 每次调用都产生一个全世界唯一、互不相等的值，常用于给对象设置"不会撞名"的特殊属性。第 7 章会展开，现在知道有这么个东西即可。

## 1.3 字符串详解：模板字符串与转义

字符串是我们最常打交道的类型，值得单独一节。

```javascript
// 文件：code/ch01/03-strings.js
// 运行方式：node 03-strings.js

const name = "小明";
const age = 18;

// ===== 老办法：用 + 拼接 =====
const s1 = "我叫" + name + "，今年" + age + "岁。";
console.log(s1);

// ===== 新办法：模板字符串（反引号 + ${}）=====
const s2 = `我叫${name}，今年${age}岁。`;
console.log(s2);

// ${...} 里可以放任意表达式，不只是变量：
console.log(`明年我${age + 1}岁。`);

// 模板字符串天然支持换行：
const poem = `静夜思
床前明月光，
疑是地上霜。`;
console.log(poem);

// ===== 常用转义字符 =====
console.log("换行：第一行\n第二行"); // \n 换行
console.log("制表符：姓名\t年龄"); // \t 制表符（对齐）
console.log("双引号里放双引号：\"引用\""); // \" 转义引号
console.log("反斜杠本身：C:\\Users"); // \\ 表示一个反斜杠

// ===== 几个常用属性/方法（第 3 章详解）=====
const word = "JavaScript";
console.log("长度：", word.length); // 10
console.log("转大写：", word.toUpperCase()); // JAVASCRIPT
console.log("截取：", word.slice(0, 4)); // Java
```

逐段讲解：

- **`+` 拼接（老办法）**：`"我叫" + name + ...`。`+` 号遇到字符串时，作用是把它们"接"在一起。这种写法一多就会淹没在一堆引号和加号里，容易出错。

- **模板字符串（template literal，新办法）**：注意 `s2` 用的不是普通引号，而是**反引号 `` ` ``**（键盘上一般在数字 `1` 左边、`Esc` 下面那个键）。反引号里可以用 `${...}` 直接嵌入变量，一眼就能看出最终长什么样。**这是现代 JS 拼字符串的首选方式**，本书后面会大量使用。

- **`${age + 1}`**：`${}` 里放的不只是变量名，而是**任意表达式**，JS 会先算出结果再塞进字符串。这里算出 `19` 再拼进去。

- **天然换行**：模板字符串里直接按回车换行，输出就真的换行，写多行文本（如上面的诗）特别方便。普通引号字符串则不能直接换行。

- **转义字符**：有些字符没法直接打出来，就用反斜杠 `\` 开头的"转义序列"表示。常见的有：`\n`（换行）、`\t`（制表符，用于对齐）、`\"`（在双引号字符串里表示一个双引号）、`\\`（表示一个反斜杠本身）。

- **`.length`、`.toUpperCase()`、`.slice()`**：字符串自带很多好用的属性和方法。`.length` 是属性（取长度，不加括号）；`.toUpperCase()` 和 `.slice()` 是方法（要加括号调用）。`slice(0, 4)` 表示"从第 0 位截到第 4 位（不含第 4 位）"，得到 `"Java"`。这些第 3 章会系统讲，这里先感受一下。

💡 从今往后，**能用模板字符串就别用 `+` 拼**。你的代码会清爽很多。

## 1.4 真假值（truthy / falsy）

前面说布尔只有 `true`/`false`。但 JavaScript 有个重要特性：**在需要布尔的地方（比如 `if` 判断里），任何值都会被自动"当作"真或假来看待。** 这叫"真值（truthy）"和"假值（falsy）"。

这套规则很关键，因为你以后会经常写 `if (用户名)`、`if (列表)` 这样"直接拿一个非布尔值来判断"的代码。

**记住这个铁律：只有下面这 6 个值是"假值（falsy）"，其余一切都是"真值（truthy）"。**

| 假值 | 说明 |
| --- | --- |
| `false` | 布尔假本身 |
| `0` | 数字零（也包括 `-0`） |
| `""` | 空字符串 |
| `null` | 主动置空 |
| `undefined` | 未赋值 |
| `NaN` | 非数字 |

📌 **反过来记更省事：背下这 6 个假值，看到不在名单里的值，一律是真值。** 特别注意几个"看起来像假、其实是真"的坑：字符串 `"0"`（里面有个字符）、字符串 `"false"`、空数组 `[]`、空对象 `{}`——**它们统统是真值！** 这些在下一节的代码里会亲眼看到。

我们会在 1.6 讲 `Boolean()` 时用代码把这 6 个假值全部验证一遍。

## 1.5 typeof 运算符

想知道某个值到底是什么类型，用 **`typeof`** 运算符。它返回一个描述类型的字符串。回看 `02-types.js` 结尾那段：

```javascript
console.log(typeof 42); // "number"
console.log(typeof "hi"); // "string"
console.log(typeof true); // "boolean"
console.log(typeof undefined); // "undefined"
console.log(typeof 10n); // "bigint"
console.log(typeof Symbol()); // "symbol"
console.log(typeof null); // "object"（历史遗留 bug，见正文）
console.log(typeof console.log); // "function"
```

大部分结果都很符合直觉，但有两个要特别说明：

⚠️ **`typeof null === "object"` 是个著名的历史遗留 bug。** 从含义上讲，`null` 明明是"空值"，`typeof` 却说它是 `"object"`。这是 1995 年 JavaScript 诞生时代码实现留下的错误，后来因为**已经有海量网页依赖这个行为**，一旦修正就会让无数老网站崩溃，所以官方决定"将错就错"、永久保留。你只需**记住这个特例**：判断一个值是不是 `null`，不要用 `typeof`，要直接写 `x === null`。

💡 **`typeof 函数` 返回 `"function"`**。严格来说函数在 JS 里属于对象的一种，但 `typeof` 特地为它返回 `"function"`，方便我们判断"这东西能不能被调用"。函数是第 2 章的主角。

`typeof` 在调试时非常有用：当程序行为诡异，很可能是某个变量的类型和你以为的不一样，`console.log(typeof x)` 一打就真相大白。

## 1.6 运算符大全

运算符（operator）就是 `+`、`-`、`===`、`&&` 这些"对数据做操作"的符号。我们分类过一遍。

```javascript
// 文件：code/ch01/04-operators.js
// 运行方式：node 04-operators.js

// ===== 算术运算符 =====
console.log("加：", 7 + 3); // 10
console.log("减：", 7 - 3); // 4
console.log("乘：", 7 * 3); // 21
console.log("除：", 7 / 3); // 2.333...
console.log("取余：", 7 % 3); // 1（余数，常用来判断奇偶）
console.log("幂：", 2 ** 10); // 1024（2 的 10 次方）

// ===== 赋值运算符（简写）=====
let n = 10;
n += 5; // 等价于 n = n + 5
console.log("n += 5 ->", n); // 15
n *= 2; // 等价于 n = n * 2
console.log("n *= 2 ->", n); // 30

// ===== 比较：== 与 === 的区别（重点！）=====
console.log("1 == '1' ?", 1 == "1"); // true：先转类型再比较（宽松相等）
console.log("1 === '1' ?", 1 === "1"); // false：类型不同直接判 false（严格相等）
console.log("0 == false ?", 0 == false); // true（隐式转换的坑）
console.log("0 === false ?", 0 === false); // false
console.log("null == undefined ?", null == undefined); // true（特例）
console.log("null === undefined ?", null === undefined); // false

// ===== 逻辑运算符与短路求值 =====
console.log("true && false ->", true && false); // false
console.log("true || false ->", true || false); // true
console.log("!true ->", !true); // false

// && 返回第一个"假值"或最后一个值；|| 返回第一个"真值"或最后一个值
console.log("'' || '默认名' ->", "" || "默认名"); // 默认名
console.log("'张三' && '已登录' ->", "张三" && "已登录"); // 已登录

// ===== 三元运算符 =====
const score = 72;
const result = score >= 60 ? "及格" : "不及格";
console.log("三元判断：", result); // 及格

// ===== ?? 空值合并 =====
const count = 0;
console.log("0 || 5 ->", count || 5); // 5（0 被当成假值，不理想）
console.log("0 ?? 5 ->", count ?? 5); // 0（?? 只在 null/undefined 时取右边）

// ===== ?. 可选链（简介）=====
const user = { name: "小红" }; // 没有 address 属性
console.log("user.address?.city ->", user.address?.city); // undefined，而不是报错
```

### 算术运算符

`+ - * /` 就是加减乘除。两个稍生的：

- **`%` 取余（取模）**：`7 % 3` 得 `1`，即 7 除以 3 的**余数**。它超级实用——判断奇偶就看 `n % 2` 是 0（偶）还是 1（奇）；每隔 5 次做点什么就看 `i % 5 === 0`。
- **`**` 幂运算**：`2 ** 10` 就是 2 的 10 次方 = 1024。

### 赋值运算符

`=` 是基本赋值。`+= -= *= /= %=` 是"简写"：`n += 5` 完全等价于 `n = n + 5`——"取出 `n` 现在的值，加上 5，再存回 `n`"。这类简写在循环计数时随处可见。

### 比较运算符：`==` vs `===`（本章最重要的知识点之一）

比较运算返回布尔值。`> < >= <=` 好理解，麻烦的是"相等"的判断，JS 有**两套**：

- **`===` 严格相等（strict equality）**：先看类型，**类型不同直接判 `false`**，类型相同再比值。`1 === "1"` 因为一个是 number、一个是 string，类型就不同，结果 `false`。
- **`==` 宽松相等（loose equality）**：如果两边类型不同，它会**先偷偷把它们转成同一类型再比**。`1 == "1"` 会把字符串 `"1"` 转成数字 `1`，于是结果 `true`。

这个"偷偷转换"看着方便，实则是**无数 bug 的源头**，因为它的规则又多又绕：`0 == false` 是 `true`、`"" == false` 是 `true`、`null == undefined` 是 `true`……几乎没人能完整记住。

📌 **黄金法则：永远用 `===` 和 `!==`，忘掉 `==` 和 `!=`。** 这是整个 JS 社区的共识，也是各种代码检查工具的默认规则。唯一常见的例外是 `x == null`（同时判断 `null` 和 `undefined`），但初学阶段你连这个都不用记，一律 `===` 就对了。

### 逻辑运算符与短路求值

- **`&&`（与 / AND）**：两边都为真才为真。
- **`||`（或 / OR）**：只要有一边为真就为真。
- **`!`（非 / NOT）**：真假反转，`!true` 是 `false`。

这里有个深一层、也非常实用的机制叫**短路求值（short-circuit evaluation）**：`&&` 和 `||` 其实**不一定返回 `true`/`false`，而是返回其中一个"操作数"本身**，并且会"能确定结果就不再往下算"。

- **`||`**：从左到右找**第一个真值**并返回它；全是假值就返回最后一个。所以 `"" || "默认名"` 中，`""` 是假值，于是返回 `"默认名"`。这就是给变量设默认值的经典写法：`const displayName = inputName || "游客";`。
- **`&&`**：从左到右，遇到**第一个假值**就返回它；全是真值就返回最后一个。`"张三" && "已登录"` 里两个都是真值，返回最后一个 `"已登录"`。它常用来"前面成立才做后面"。

"短路"的意思是：`false && 危险操作()` 里，因为 `&&` 左边已是 `false`，结果必然为假，右边的 `危险操作()` **根本不会执行**。这个特性以后会反复用到。

### 三元运算符

`条件 ? 值A : 值B` 是唯一需要三个部分的运算符，叫**三元运算符**。读作"如果`条件`为真，取`值A`，否则取`值B`"。`score >= 60 ? "及格" : "不及格"` 就是一句话版的 `if/else`，特别适合"根据条件二选一"的场景。

### `??` 空值合并运算符

`a ?? b` 的意思是："如果 `a` 是 `null` 或 `undefined`，就取 `b`，否则取 `a`。"

它和 `||` 长得像，但有个关键区别：`||` 对**所有假值**都会取右边，而 `??` **只对 `null`/`undefined`** 取右边。看代码里 `count = 0` 的对比：`0 || 5` 得 `5`（因为 `0` 是假值，但这往往不是我们想要的——`0` 明明是个有效数字！），而 `0 ?? 5` 得 `0`（因为 `0` 不是 `null`/`undefined`，保留原值）。

📌 **当"0、空字符串、false 都是合法值，你只想在'真的没值'时才用默认值"时，用 `??` 而不是 `||`。** 这是现代 JS 处理默认值的更精确方式。

### `?.` 可选链（简介）

`user.address?.city` 中的 `?.` 叫**可选链（optional chaining）**。它的作用是：如果 `?.` 左边是 `null`/`undefined`，整个表达式**直接返回 `undefined`，而不是报错崩溃**。

对象和属性访问是第 3 章的内容，这里只需知道：读取"可能不存在的深层属性"时，`?.` 能帮你避免 "Cannot read property of undefined" 这个新手最常见的报错。等学到对象你就会天天用它。

## 1.7 类型转换：显式与隐式

JavaScript 是**动态类型（dynamically typed）**语言——同一个变量，前一刻装数字、后一刻装字符串都行，类型是在运行时才确定的。这带来灵活，也带来了"类型转换"这个必须搞懂的主题。转换分两种：**显式**（你亲手调用函数去转）和**隐式**（JS 在背后偷偷帮你转）。

```javascript
// 文件：code/ch01/05-conversion.js
// 运行方式：node 05-conversion.js

// ===== 显式转换：你主动调用函数 =====

// 1) 转数字 Number()
console.log("Number('42') ->", Number("42")); // 42
console.log("Number('3.14') ->", Number("3.14")); // 3.14
console.log("Number('') ->", Number("")); // 0（空字符串转成 0！）
console.log("Number('abc') ->", Number("abc")); // NaN（转不了就是 NaN）
console.log("Number(true) ->", Number(true)); // 1
console.log("Number(false) ->", Number(false)); // 0
console.log("Number(null) ->", Number(null)); // 0
console.log("Number(undefined) ->", Number(undefined)); // NaN

// parseInt / parseFloat：从字符串"开头"能读多少读多少
console.log("parseInt('42px') ->", parseInt("42px")); // 42
console.log("parseFloat('3.14m') ->", parseFloat("3.14m")); // 3.14

// 2) 转字符串 String()
console.log("String(42) ->", String(42)); // "42"
console.log("String(true) ->", String(true)); // "true"
console.log("String(null) ->", String(null)); // "null"

// 3) 转布尔 Boolean()
console.log("Boolean(1) ->", Boolean(1)); // true
console.log("Boolean(0) ->", Boolean(0)); // false
console.log("Boolean('') ->", Boolean("")); // false
console.log("Boolean('hi') ->", Boolean("hi")); // true

// ===== 假值（falsy）全家福：只有这 6 个会被当成 false =====
console.log("--- 6 个假值 ---");
console.log(Boolean(false)); // false
console.log(Boolean(0)); // false
console.log(Boolean("")); // false
console.log(Boolean(null)); // false
console.log(Boolean(undefined)); // false
console.log(Boolean(NaN)); // false
// 其余一切（包括 "0"、"false"、[]、{}）都是真值（truthy）
console.log('Boolean("0") ->', Boolean("0")); // true！字符串 "0" 是真值
console.log("Boolean([]) ->", Boolean([])); // true！空数组是真值

// ===== 隐式转换：JS 偷偷帮你转（陷阱重灾区）=====
console.log("--- 隐式转换 ---");
console.log("'5' + 3 ->", "5" + 3); // "53"：+ 遇到字符串就变拼接
console.log("'5' - 3 ->", "5" - 3); // 2：- 只能算数，把 '5' 转成数字
console.log("'5' * '2' ->", "5" * "2"); // 10：都转成数字
console.log("true + 1 ->", true + 1); // 2：true 转成 1
console.log("[] + [] ->", [] + []); // ""（空字符串，很反直觉）
```

### 显式转换：三个转换函数

- **`Number(x)`**：把 `x` 转成数字。注意几个坑：`Number("")` 是 `0`（空字符串变零！）；`Number("abc")` 转不了就是 `NaN`；`Number(true)` 是 `1`、`Number(false)` 是 `0`；`Number(null)` 是 `0` 而 `Number(undefined)` 是 `NaN`（这俩"空"转出来还不一样）。
- **`parseInt` / `parseFloat`**：它俩更宽容，从字符串**开头**尽量读数字，读到非数字就停。`parseInt("42px")` 得 `42`，`parseFloat("3.14m")` 得 `3.14`。处理 "100px" 这种带单位的文本时很好用。
- **`String(x)`**：把任何值转成字符串。`String(42)` 得 `"42"`，`String(null)` 得 `"null"`。
- **`Boolean(x)`**：把任何值转成布尔——这正好用来验证 1.4 讲的真假值规则！代码里 `Boolean(false/0/""/null/undefined/NaN)` 六个全是 `false`，其余都是 `true`。特别留意最后两行：`Boolean("0")` 是 `true`（字符串 `"0"` 里有个字符，非空，所以是真值），`Boolean([])` 也是 `true`（空数组是真值）。这两个坑请务必亲手跑一遍加深印象。

### 隐式转换：JS 背着你转

真正折磨新手的是**隐式转换**——你没要求转，JS 却自作主张转了。看代码最后一段：

- **`"5" + 3` 得 `"53"`**：`+` 号很特殊，**只要有一边是字符串，它就切换成"字符串拼接"模式**，把另一边也转成字符串。于是数字 `3` 变成 `"3"`，拼出 `"53"`。
- **`"5" - 3` 得 `2`**：而 `-`（以及 `*` `/` `%`）**没有拼接含义，只能做算术**，于是反过来把字符串 `"5"` 转成数字 `5`，算出 `2`。
- **`true + 1` 得 `2`**：`true` 被转成 `1`。
- **`[] + []` 得 `""`**：空数组转成空字符串后拼接，得到空字符串——极其反直觉，纯粹是"隐式转换太随意"的反面教材。

📌 **实务建议：不要依赖隐式转换，尤其从用户输入或命令行读来的数据，一律先用 `Number()` / `String()` 显式转成你想要的类型再用。** 本章小项目里，我们从命令行读到的参数天生是字符串，就必须先 `Number()` 转一下才能参与计算——这是真实开发中每天都在做的事。

### 埋个伏笔：为什么会有 TypeScript

你可能已经隐约感到：JS 的动态类型 + 隐式转换，"自由"得有点危险。一个变量到底装的是数字还是字符串，全靠运行时才知道；`"5" - 3` 和 `"5" + 3` 结果天差地别却都不报错。程序一大，这种"类型不确定"就会埋下大量难查的 bug。

**TypeScript（TS）正是为了解决这个问题而生的**：它给 JavaScript 加上了"类型标注"，让你在**写代码时**就声明"这个变量必须是数字"，一旦你不小心传了字符串，编辑器当场标红、编译报错，把 bug 消灭在运行之前。本书第四部分（第 13 章起）会带你系统学习 TS。现在只需记住：**你今天在 JS 里踩过的每一个类型坑，都是将来爱上 TypeScript 的理由。**

## 常见坑与最佳实践

把本章的"血泪教训"集中列一遍，建议做完项目回头再看一遍：

- ⚠️ **别用 `==`，一律用 `===`。** `==` 的隐式转换规则复杂又反直觉，是 bug 温床。
- ⚠️ **别直接用 `===` 比较两个小数。** 因浮点精度，`0.1 + 0.2 === 0.3` 是 `false`。需要时先 `.toFixed()` 或比较差值。
- ⚠️ **判断 `NaN` 不能用 `=== NaN`**（它谁都不等），要用 `Number.isNaN(x)`。
- ⚠️ **`typeof null` 是 `"object"`**，历史遗留。判断 `null` 请写 `x === null`。
- ⚠️ **`"0"`、`"false"`、`[]`、`{}` 都是真值**，只有那 6 个才是假值。别想当然。
- ⚠️ **命令行 / 用户输入拿到的永远是字符串**，参与数学运算前务必 `Number()` 转换。
- 📌 **默认用 `const`，需要改再换 `let`，永远别用 `var`。**
- 📌 **拼字符串优先用模板字符串**（反引号 + `${}`），别再 `+` 一大串。
- 📌 **设默认值时，能用 `??` 就别用 `||`**——除非你确实想让 `0`/`""` 也走默认值。
- 💡 **变量名要见名知意**，用 camelCase；调试时多用 `console.log(typeof x)` 和 `console.log(x)`。

## 本章小项目：单位/汇率换算器

### 需求

做一个命令行小工具，能完成两类换算：

1. **温度**：摄氏 ↔ 华氏互转。
2. **货币**：美元 ↔ 人民币互转（用一个写死的固定汇率）。

要求既能"不带参数直接跑，看几个演示"，也能"带命令行参数，算用户想算的"。整个项目要用上本章的：`const` 常量、number 计算、模板字符串、类型转换（`Number()`）、`NaN` 校验、比较与判断。

### 设计思路

一步步想清楚"程序要干什么"，再动手写：

1. **换算公式**是固定的数学关系，我们把每种换算写成一个小函数（函数是第 2 章的内容，这里先用起来，你会发现很自然）。温度公式：华氏 `F = C × 9/5 + 32`，反过来 `C = (F − 32) × 5/9`。
2. **汇率**是个不该被中途改动的标准值，用 `const` 存。
3. **怎么拿到用户要算的数？** 用 `process.argv`——这是 Node 提供的数组，存着你在命令行敲的参数。关键点：**从命令行读来的东西全是字符串**，`"100"` 而不是数字 `100`，所以算之前必须 `Number()` 转一下，还要用 `Number.isNaN()` 挡住 `"abc"` 这种乱输入。
4. 如果用户**什么参数都没给**，就打印几个内置示例——这样零基础的你直接 `node converter.js` 就能看到效果，不必先学命令行参数。
5. 结果统一保留两位小数（`toFixed(2)`），用模板字符串拼成好看的句子输出。

### 完整实现

```javascript
// 文件：code/ch01/converter.js
// 单位/汇率换算器 —— 第 1 章小项目
// 运行方式（默认示例）：node converter.js
// 运行方式（命令行参数）：
//   node converter.js c2f 100      -> 摄氏 100 度转华氏
//   node converter.js f2c 212      -> 华氏 212 度转摄氏
//   node converter.js usd2cny 20   -> 20 美元转人民币
//   node converter.js cny2usd 145  -> 145 人民币转美元

// ===== 1. 固定配置（用 const，因为这些"标准"不该被程序中途改掉）=====
const USD_TO_CNY = 7.25; // 假设 1 美元 = 7.25 人民币（示例用固定汇率）

// ===== 2. 四个换算公式，各写成一句表达式 =====
// 摄氏 -> 华氏： F = C × 9/5 + 32
function celsiusToFahrenheit(c) {
  return (c * 9) / 5 + 32;
}
// 华氏 -> 摄氏： C = (F − 32) × 5/9
function fahrenheitToCelsius(f) {
  return ((f - 32) * 5) / 9;
}
// 美元 -> 人民币
function usdToCny(usd) {
  return usd * USD_TO_CNY;
}
// 人民币 -> 美元
function cnyToUsd(cny) {
  return cny / USD_TO_CNY;
}

// 把数字保留两位小数，返回字符串（便于展示金额/温度）
function round2(n) {
  return n.toFixed(2);
}

// ===== 3. 读取命令行参数 =====
// process.argv 是一个数组：
//   [0] node 可执行文件路径
//   [1] 当前脚本路径
//   [2] 起才是我们真正传的参数
const op = process.argv[2]; // 操作类型，如 "c2f"
const rawValue = process.argv[3]; // 数值（此时还是字符串！）

// ===== 4. 没给参数时，跑几个内置示例（对零基础最友好）=====
if (op === undefined) {
  console.log("=== 单位 / 汇率换算器（示例演示）===");
  console.log(`摄氏 100°C = 华氏 ${round2(celsiusToFahrenheit(100))}°F`);
  console.log(`华氏 212°F = 摄氏 ${round2(fahrenheitToCelsius(212))}°C`);
  console.log(`20 美元   = ${round2(usdToCny(20))} 人民币`);
  console.log(`145 人民币 = ${round2(cnyToUsd(145))} 美元`);
  console.log("");
  console.log("想自己算？试试：node converter.js c2f 37");
} else {
  // ===== 5. 给了参数：先把字符串转成数字并校验 =====
  const value = Number(rawValue); // 关键：命令行参数永远是字符串，必须转

  if (Number.isNaN(value)) {
    console.log(`⚠️ "${rawValue}" 不是一个有效数字。`);
  } else {
    // 用一串三元/判断根据 op 选择公式
    let output;
    if (op === "c2f") {
      output = `摄氏 ${value}°C = 华氏 ${round2(celsiusToFahrenheit(value))}°F`;
    } else if (op === "f2c") {
      output = `华氏 ${value}°F = 摄氏 ${round2(fahrenheitToCelsius(value))}°C`;
    } else if (op === "usd2cny") {
      output = `${value} 美元 = ${round2(usdToCny(value))} 人民币`;
    } else if (op === "cny2usd") {
      output = `${value} 人民币 = ${round2(cnyToUsd(value))} 美元`;
    } else {
      output = `⚠️ 不认识的操作 "${op}"。可用：c2f / f2c / usd2cny / cny2usd`;
    }
    console.log(output);
  }
}
```

### 逐段讲解

1. **`const USD_TO_CNY = 7.25;`** —— 汇率用 `const` 存，全大写命名表明它是"配置常量"。用 `const` 能防止程序别处不小心把它改掉。

2. **四个换算函数** —— 每个函数把一条数学公式包起来。`function 名字(参数) { return 结果; }` 是函数的基本形态（第 2 章细讲）。这里注意运算优先级：`(c * 9) / 5 + 32`，先算乘除、再算加，和数学一致；`((f - 32) * 5) / 9` 用括号确保先减 32。把公式拆成函数的好处是——**每个换算只写一次，哪里要用就调用，改公式也只改一处**。

3. **`round2(n)`** —— `n.toFixed(2)` 把数字四舍五入保留两位小数。⚠️ 注意它返回的是**字符串**（如 `"212.00"`）不是数字，这正好方便我们直接拼进结果文本。这也是应对 1.2 那个"浮点精度"坑的实用手段。

4. **`process.argv`** —— 这是本项目的核心知识点。当你运行 `node converter.js c2f 100`，Node 会把命令行拆成一个数组存进 `process.argv`：第 `[0]` 个是 node 程序路径，第 `[1]` 个是脚本路径，**从第 `[2]` 个起才是你真正传的参数**。所以 `op = process.argv[2]` 拿到 `"c2f"`，`rawValue = process.argv[3]` 拿到 `"100"`。请牢牢记住：**这个 `"100"` 是字符串，不是数字！**

5. **`if (op === undefined)`** —— 如果第 `[2]` 个参数不存在，`process.argv[2]` 就是 `undefined`，说明用户没带参数，我们走"演示分支"，打印四个写死的示例。注意这里用 `===`（严格相等）判断，符合本章的黄金法则。示例输出全用模板字符串拼接，`${round2(...)}` 把函数结果直接嵌进句子。

6. **`const value = Number(rawValue);`** —— 用户带了参数时，第一件事就是把字符串 `rawValue` 用 `Number()` **显式转成数字**。这正是 1.7 强调的"用户输入必须先转类型"。

7. **`if (Number.isNaN(value))`** —— 转换可能失败：用户要是输入 `"abc"`，`Number("abc")` 就是 `NaN`。我们用 `Number.isNaN()`（而不是 `=== NaN`，因为 `NaN` 谁都不等）来检测并友好提示，避免后面算出一堆 `NaN`。这就是 1.2 那个坑在真实项目里的应对。

8. **`if / else if` 选公式** —— 根据 `op` 的值决定调用哪个换算函数，把结果拼成句子存进 `output`，最后统一 `console.log`。最后的 `else` 兜底处理"用户输错了操作名"的情况。

```bash
node converter.js c2f abc      # ⚠️ "abc" 不是一个有效数字。
```

## 小结与自测

**本章重点回顾：**

- **变量**是"贴了名字的盒子"。用 `const`（默认）和 `let`（需要改时）声明，**别用 `var`**。`=` 是赋值不是相等。
- **7 种原始类型**：number、string、boolean、undefined、null、bigint、symbol。number 不分整数小数，但有浮点精度、`NaN`、`Infinity` 三个坑。
- **模板字符串**（反引号 + `${}`）是拼文本的首选，还支持换行；`\n \t \" \\` 是常用转义。
- **6 个假值**：`false 0 "" null undefined NaN`，其余全是真值（`"0"`、`[]`、`{}` 都是真值！）。
- **`typeof`** 查类型；牢记 `typeof null === "object"` 是历史遗留特例。
- **运算符**：`%` 取余、`**` 幂；比较**只用 `===`/`!==`**；逻辑运算有**短路求值**，`||`/`&&` 返回操作数本身；`? :` 三元、`??` 空值合并、`?.` 可选链。
- **类型转换**：显式用 `Number()`/`String()`/`Boolean()`，隐式转换（如 `"5" + 3`）是陷阱重灾区，**用户输入务必先显式转换**。
- JS 是**动态类型**语言，灵活但易埋类型 bug——这正是后面 TypeScript 要解决的痛点。
