## 5.1 从工厂函数说起：面向对象在解决什么

在第 3 章我们学过对象字面量，可以手写一个对象：

```javascript
const alice = {
  name: "Alice",
  age: 30,
  greet() {
    return `你好，我叫 ${this.name}`;
  },
};
```

但如果要造 100 个用户呢？总不能手写 100 遍。自然的想法是写一个函数，输入数据、返回对象——这就是 **工厂函数（factory function）**，像工厂流水线一样批量生产对象。

```javascript
// 文件：code/ch05/01-factory.js
function createUser(name, age) {
  return {
    name,
    age,
    greet() {
      return `你好，我叫 ${this.name}，今年 ${this.age} 岁。`;
    },
  };
}

const alice = createUser("Alice", 30);
const bob = createUser("Bob", 25);

console.log(alice.greet()); // 你好，我叫 Alice，今年 30 岁。
console.log(bob.greet()); // 你好，我叫 Bob，今年 25 岁。

console.log("两个 greet 是同一个函数吗？", alice.greet === bob.greet); // false
```

**逐段拆解：**

- `createUser` 每次被调用，都 `return` 一个**全新的对象字面量**。`{ name, age }` 用了第 3 章的属性简写。
- `alice` 和 `bob` 各是一个独立对象，`greet()` 里的 `this` 指向调用它的那个对象（第 4 章的隐式绑定规则），所以打印各自的名字。
- 最后一行是**关键**：`alice.greet === bob.greet` 是 `false`。

📌 这暴露了工厂函数的**局限**：每造一个对象，就**重新创建了一份 `greet` 函数**。方法体明明一模一样，却在内存里存了两份。造一万个用户，就有一万份重复的 `greet`，纯属浪费。

我们真正想要的是：**数据（`name`、`age`）每个对象各有一份，但方法（`greet`）大家共享同一份。** JavaScript 用一种独特的机制来实现"共享"——**原型（prototype）**。这就是我们接下来要攻克的核心概念。

## 5.2 原型与原型链：对象背后的"备用对象"

### 每个对象都有一个"原型"

在 JavaScript 里，**几乎每个对象都有一个隐藏的关联对象，叫作它的原型（prototype）**。你可以把原型想象成一个"备用对象"或"后备箱"：当你想访问某个对象的属性时，如果它**自己身上没有**，JavaScript 就会去它的原型上找；原型上也没有，就去原型的原型上找……如此形成一条链，直到找到或到头为止。这条链就是 **原型链（prototype chain）**。

打个类比：你去公司前台找一份文件（属性），前台自己没有（对象自身没有），就打电话问二楼的部门（原型），二楼没有再问三楼（原型的原型）……一层层往上问，直到问到或到顶。

### 用 `Object.create` 手动指定原型

最直白的方式是用 `Object.create(proto)`，它创建一个新对象，并把它的原型指定为 `proto`：

```javascript
// 文件：code/ch05/02-prototype.js
const animal = {
  eats: true,
  walk() {
    return `${this.name} 正在走路`;
  },
};

const rabbit = Object.create(animal); // rabbit 的原型是 animal
rabbit.name = "兔子";
rabbit.jumps = true;

console.log(rabbit.name); // 兔子（自己的属性）
console.log(rabbit.jumps); // true（自己的属性）
console.log(rabbit.eats); // true（自己没有，沿原型链在 animal 上找到）
console.log(rabbit.walk()); // 兔子 正在走路
```

**逐段拆解：**

- `animal` 是一个普通对象，有属性 `eats` 和方法 `walk`。
- `Object.create(animal)` 创建了 `rabbit`，并把 `rabbit` 的原型设为 `animal`。此刻 `rabbit` 自己身上什么都没有。
- 我们给 `rabbit` 加了 `name` 和 `jumps`，这两个是它**自己的属性**。
- 读 `rabbit.eats` 时，`rabbit` 自己没有 `eats`，于是沿原型链到 `animal` 找到了 `true`。
- 最有意思的是 `rabbit.walk()`：`walk` 定义在 `animal` 上，但它内部的 `this` 指向**调用它的那个对象**，也就是 `rabbit`。所以打印的是"兔子 正在走路"，而不是"undefined 正在走路"。

📌 记住这条黄金规则：**方法在哪个对象上定义不重要，`this` 指向的是"点号左边"那个调用者。** 这正是"方法共享、数据独立"的秘密——`walk` 只有一份（在 `animal` 上），但每个继承它的对象调用时，`this` 各自不同。

### 查看和检查原型

```javascript
// 文件：code/ch05/02-prototype.js（续）
console.log(Object.getPrototypeOf(rabbit) === animal); // true
console.log(rabbit.__proto__ === animal); // true（等价，但不推荐用）
```

- `Object.getPrototypeOf(obj)` 是**推荐**的、获取一个对象原型的标准方式。
- `obj.__proto__`（读作 "dunder proto"）是一个历史遗留的访问器，效果相同。

⚠️ **陷阱**：`__proto__` 虽然到处能用，但它是早期浏览器的非标准产物，性能差、语义混乱。**代码里请用 `Object.getPrototypeOf` / `Object.setPrototypeOf`**，`__proto__` 只在调试时随手看看即可。

### 原型链是多层的

原型的原型还可以有原型，层层相连：

```javascript
// 文件：code/ch05/02-prototype.js（续）
const longEar = Object.create(rabbit);
longEar.name = "长耳兔";
console.log(longEar.eats); // true（跨了两层去找）
```

`longEar` 的原型是 `rabbit`，`rabbit` 的原型是 `animal`。读 `longEar.eats` 时：`longEar` 没有 → `rabbit` 没有 → `animal` 有，返回 `true`。这条链是：

```
longEar → rabbit → animal → Object.prototype → null
```

链的尽头是 `Object.prototype`（所有普通对象的"祖先"，提供了 `hasOwnProperty`、`toString` 等），再往上就是 `null`，表示到顶了。

### `hasOwnProperty` 与 `in`：自己的还是继承的？

有时我们需要区分：某个属性是对象**自己的**，还是**从原型继承来的**？

```javascript
// 文件：code/ch05/02-prototype.js（续）
console.log(rabbit.hasOwnProperty("name")); // true（自己的）
console.log(rabbit.hasOwnProperty("eats")); // false（继承来的，不算自己的）

console.log("eats" in rabbit); // true（in 会连原型链一起看）
```

- `obj.hasOwnProperty(key)`：**只看对象自己身上**有没有，不看原型链。`name` 是 `rabbit` 自己加的，返回 `true`；`eats` 来自 `animal`，返回 `false`。
- `key in obj`：**连原型链一起看**。`eats` 虽然是继承的，但沿链能找到，返回 `true`。

💡 **技巧**：遍历对象自身属性、避免把继承来的属性也算进去时，`hasOwnProperty` 非常有用。更现代的写法是 `Object.hasOwn(obj, key)`（ES2022），语义相同但更安全，推荐使用。

### 属性遮蔽（shadowing）

如果给对象自己加一个和原型同名的属性，会发生什么？

```javascript
// 文件：code/ch05/02-prototype.js（续）
rabbit.eats = false; // 在 rabbit 自己身上新建了 eats
console.log(rabbit.eats); // false（自己的遮蔽了原型的）
console.log(animal.eats); // true（原型没被改动）
```

**逐段拆解：**赋值 `rabbit.eats = false` **不会**去修改原型上的 `eats`，而是在 `rabbit` **自己身上**新建一个 `eats`。此后读 `rabbit.eats` 先在自己身上找到 `false`，就不再往上查了——这叫 **属性遮蔽（property shadowing）**。原型上的 `animal.eats` 依然是 `true`，安然无恙。

📌 记住：**读属性会沿原型链查找，写属性只写在对象自己身上**（除非涉及 setter，后面会讲）。

## 5.3 构造函数与 new：语法糖的前身

`Object.create` 能手动搭原型，但每次还要手动加属性，很啰嗦。JavaScript 早期提供了另一套惯用法：**构造函数（constructor function）+ `new`**。

### 构造函数怎么写

```javascript
// 文件：code/ch05/03-constructor-new.js
function User(name, age) {
  this.name = name;
  this.age = age;
}

User.prototype.greet = function () {
  return `你好，我叫 ${this.name}，今年 ${this.age} 岁。`;
};

const alice = new User("Alice", 30);
const bob = new User("Bob", 25);

console.log(alice.greet()); // 你好，我叫 Alice，今年 30 岁。
console.log(bob.greet()); // 你好，我叫 Bob，今年 25 岁。

console.log("共享同一个 greet？", alice.greet === bob.greet); // true
```

**逐段拆解：**

- `User` 就是一个普通函数，但**约定首字母大写**，表示它是拿来配合 `new` 用的构造函数。
- 函数体里 `this.name = name`——注意这里的 `this`，它并不是我们手动创建的对象，而是 `new` 帮我们悄悄准备好的（下面详解）。
- `User.prototype.greet = ...`：每个函数都自带一个 `prototype` 属性（一个对象）。我们把共享方法挂在上面。
- 最后 `alice.greet === bob.greet` 这次是 **`true`**！因为 `greet` 只有一份，挂在 `User.prototype` 上，所有实例共享。这正好解决了 5.1 节工厂函数的浪费问题。

⚠️ **两个容易混淆的 `prototype`**：
- **函数**的 `prototype` 属性（`User.prototype`）：是一个对象，用来放"将来实例共享的方法"。
- **对象**的原型（`Object.getPrototypeOf(alice)`）：是实例真正链接到的那个原型对象。

它们的关系是：`new User()` 造出的实例，其**原型**正是 `User.prototype`。

```javascript
// 文件：code/ch05/03-constructor-new.js（续）
console.log(Object.getPrototypeOf(alice) === User.prototype); // true
console.log(alice.constructor === User); // true
```

`User.prototype` 上还自带一个 `constructor` 属性，指回 `User` 本身，所以 `alice.constructor` 能找到 `User`。

### 📌 new 到底做了什么（四步）

这是本章的重中之重，务必记牢。当你写 `new User("Alice", 30)` 时，JavaScript 在幕后做了**四件事**：

1. **创建**一个全新的空对象。
2. 把这个新对象的**原型**指向 `User.prototype`。
3. 以这个新对象作为 `this`，**执行** `User` 函数体（于是 `this.name = name` 等把属性挂到了新对象上）。
4. 如果函数**没有显式返回一个对象**，就**自动返回**这个新对象。

为了把这四步看得清清楚楚，我们**亲手实现一个 `myNew`**：

```javascript
// 文件：code/ch05/03-constructor-new.js（续）
function myNew(Constructor, ...args) {
  // 第 1、2 步：创建新对象，其原型指向 Constructor.prototype
  const obj = Object.create(Constructor.prototype);
  // 第 3 步：以该对象为 this 执行构造函数
  const result = Constructor.apply(obj, args);
  // 第 4 步：构造函数返回对象则用它，否则用我们创建的 obj
  return typeof result === "object" && result !== null ? result : obj;
}

const carol = myNew(User, "Carol", 40);
console.log(carol.greet()); // 你好，我叫 Carol，今年 40 岁。
console.log(carol instanceof User); // true
```

**逐段拆解：**

- `Object.create(Constructor.prototype)` 一步就完成了"创建新对象 + 设定原型"（第 1、2 步）。
- `Constructor.apply(obj, args)` 用第 4 章学的 `apply` 把 `this` 显式绑定成 `obj` 并执行构造函数（第 3 步）。构造函数里的 `this.name = ...` 就把属性挂到了 `obj` 上。
- 最后判断：如果构造函数返回了一个对象（少见但合法），就返回那个对象；否则返回我们造的 `obj`（第 4 步）。
- `carol` 由我们手写的 `myNew` 造出，却和真正的 `new` 效果一致：能调 `greet`，`instanceof User` 也是 `true`。

理解了这四步，`new`、`this`、`prototype` 三者的关系就彻底打通了。而现代的 `class` 语法，本质上就是把这一套包装得更好看——接下来就见分晓。

## 5.4 class 语法：原型的现代外衣

构造函数 + `prototype` 的写法能用，但零散、易错（比如忘了 `new` 会污染全局 `this`）。ES2015 引入了 **`class`（类）** 语法，让面向对象写起来更清晰。

但请**牢记一句话**：📌 **class 只是原型机制的"语法糖"（syntactic sugar）**，底层依然是原型和构造函数，没有任何新魔法。我们会在本节末尾验证这一点。

### 一个完整的 class

```javascript
// 文件：code/ch05/04-class.js
class BankAccount {
  owner = "未命名"; // 公有字段（class field）

  #balance = 0; // 私有字段（private field），只能在类内访问

  static bankName = "示例银行"; // 静态字段，属于类本身

  static #count = 0; // 静态私有字段

  constructor(owner, initial = 0) {
    this.owner = owner;
    this.#balance = initial;
    BankAccount.#count++;
  }

  deposit(amount) {
    if (amount <= 0) return this.#balance;
    this.#balance += amount;
    return this.#balance;
  }

  withdraw(amount) {
    if (amount > this.#balance) {
      console.log("余额不足");
      return this.#balance;
    }
    this.#balance -= amount;
    return this.#balance;
  }

  get balance() {
    return this.#balance;
  }

  set balance(value) {
    if (typeof value !== "number" || value < 0) {
      console.log("非法余额，已忽略");
      return;
    }
    this.#balance = value;
  }

  static totalAccounts() {
    return BankAccount.#count;
  }
}
```

这段代码信息量很大，我们逐个概念拆开讲。

**① constructor（构造方法）**

```javascript
constructor(owner, initial = 0) {
  this.owner = owner;
  this.#balance = initial;
  BankAccount.#count++;
}
```

`constructor` 是一个特殊方法，`new BankAccount(...)` 时会自动调用它，作用相当于前面的构造函数体。参数可以有默认值（`initial = 0`）。每 new 一个账户，就让静态计数 `#count` 加一。

**② 实例方法（instance method）**

`deposit`、`withdraw` 直接写在类体里，**自动挂到 `BankAccount.prototype` 上**，所有实例共享同一份——正是 5.3 节手动 `User.prototype.greet = ...` 的自动化版本。

**③ 公有字段（class field）**

```javascript
owner = "未命名";
```

直接在类体里写 `字段名 = 默认值`，相当于在 constructor 里 `this.owner = "未命名"`。它是**每个实例各自一份**的数据。

**④ 私有字段（private field）`#`**

```javascript
#balance = 0;
```

以 `#` 开头的字段是**私有的**，只能在类**内部**访问，外部完全碰不到。这是真正的封装（encapsulation）——把内部状态藏起来，只暴露受控的接口（`deposit`/`withdraw`/getter）。

```javascript
// 文件：code/ch05/04-class.js（使用）
const acc = new BankAccount("Alice", 100);
// console.log(acc.#balance); // ⚠️ SyntaxError：私有字段外部不可访问
```

⚠️ **陷阱**：`#balance` 里的 `#` 是名字的**一部分**，不是运算符。而且它必须先在类体里声明才能用。在类外访问 `acc.#balance` 会直接**语法报错**，不是返回 `undefined`。

**⑤ getter / setter（存取器）**

```javascript
get balance() {
  return this.#balance;
}
set balance(value) {
  if (typeof value !== "number" || value < 0) {
    console.log("非法余额，已忽略");
    return;
  }
  this.#balance = value;
}
```

getter/setter 让你**像读写普通属性一样**，背后却执行方法：

```javascript
// 文件：code/ch05/04-class.js（使用）
acc.deposit(50);
console.log(acc.balance); // 150（读 balance 走的是 getter）
acc.balance = 999; // 赋值走的是 setter
console.log(acc.balance); // 999
acc.balance = -1; // 非法余额，已忽略（setter 拦截了）
console.log(acc.balance); // 999
```

注意用法：**没有括号**！`acc.balance` 而不是 `acc.balance()`。getter/setter 常用来在读写时做校验、计算或格式化，同时对外隐藏 `#balance` 的存在。

**⑥ 静态成员（static）**

```javascript
static bankName = "示例银行";
static totalAccounts() {
  return BankAccount.#count;
}
```

带 `static` 的字段和方法属于**类本身**，而不是某个实例。它们通过**类名**调用：

```javascript
// 文件：code/ch05/04-class.js（使用）
console.log(BankAccount.bankName); // 示例银行
new BankAccount("Bob");
console.log(BankAccount.totalAccounts()); // 2
```

`bankName` 是所有账户共享的银行名（不属于某个账户），`totalAccounts()` 是一个统计工具。静态方法里访问不到实例的 `this.xxx`，因为它跟具体实例无关。静态成员常用于工具函数、工厂方法、常量、计数器。

### 验证：class 真的只是语法糖

```javascript
// 文件：code/ch05/04-class.js（验证）
console.log(typeof BankAccount.prototype.deposit); // function
console.log(Object.getPrototypeOf(acc) === BankAccount.prototype); // true
```

看：`deposit` 确实挂在 `BankAccount.prototype` 上，实例 `acc` 的原型就是 `BankAccount.prototype`——和 5.3 节手写构造函数的结构一模一样。**class 没有引入新的对象模型，它只是把原型那套写法变得更清爽、更安全。**

💡 **class 相比构造函数的几个额外好处**：类体内的代码自动处于严格模式；方法不可枚举（不会在 `for...in` 里意外冒出来）；忘记写 `new` 直接调用 class 会**报错**（构造函数则会静默污染全局），更安全。

## 5.5 继承：extends 与 super

面向对象的另一大支柱是**继承（inheritance）**：让一个类"接过"另一个类的属性和方法，只需补充或修改差异部分，避免重复代码。

### extends：声明父子关系

```javascript
// 文件：code/ch05/05-inheritance.js
class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    return `${this.name} 发出了声音`;
  }

  describe() {
    return `这是 ${this.name}，${this.speak()}`;
  }
}

class Dog extends Animal {
  constructor(name, breed) {
    super(name); // 调用父类构造函数
    this.breed = breed;
  }

  speak() {
    return `${this.name}（${this.breed}）汪汪叫`;
  }

  describe() {
    return super.describe() + "，它是一只狗";
  }
}
```

**逐段拆解：**

- `class Dog extends Animal`：`extends` 表示 `Dog` **继承自** `Animal`，`Animal` 是**父类（superclass）**，`Dog` 是**子类（subclass）**。`Dog` 自动拥有 `Animal` 的方法。

- **`super(name)`**：在子类的 constructor 里，`super(...)` 会**调用父类的 constructor**。这里它执行 `Animal` 的构造，把 `this.name` 设好，然后子类再补上 `this.breed`。

  📌 **铁律：在子类 constructor 中，必须先调用 `super(...)`，才能使用 `this`。** 因为父类构造还没跑，`this` 还没被完整初始化，提前用会直接报错。

- **方法重写（override）**：`Dog` 定义了自己的 `speak`，**覆盖**了从 `Animal` 继承来的同名方法。这不是删除父类方法，而是"就近优先"——原型链上先找到 `Dog.prototype.speak`。

- **`super.describe()`**：在**方法**里，`super.方法名()` 调用**父类的同名方法**。`Dog` 的 `describe` 先借用父类的实现，再拼上"它是一只狗"。这样既复用父类逻辑，又能扩展。

### super 的两种用法，别混淆

⚠️ 这是初学者最容易搞错的地方：

- 在 **constructor 里**，`super(参数)` 当**函数**用，作用是**执行父类构造函数**。
- 在**普通方法里**，`super.方法名(参数)` 当**对象**用，作用是**调用父类的那个方法**。

### 运行看看，理解多态

```javascript
// 文件：code/ch05/05-inheritance.js（使用）
const generic = new Animal("某动物");
const dog = new Dog("旺财", "柴犬");
const cat = new Cat("咪咪"); // Cat 也 extends Animal，只重写了 speak

console.log(dog.speak()); // 旺财（柴犬）汪汪叫
console.log(dog.describe()); // 这是 旺财，旺财（柴犬）汪汪叫，它是一只狗
console.log(cat.describe()); // 这是 咪咪，咪咪 喵喵叫
```

特别注意 `cat.describe()`：`Cat` **没有**重写 `describe`，用的是从 `Animal` 继承来的那个。而 `Animal.describe` 内部调用了 `this.speak()`——此时 `this` 是 `cat`，于是调用的是 **`Cat` 自己的 `speak`**（"喵喵叫"），而不是 `Animal` 的 `speak`。

📌 这就是 **多态（polymorphism）**：**同一段代码（`this.speak()`），作用在不同对象上，表现出不同行为。** 父类的 `describe` 不需要知道子类具体是狗还是猫，运行时自动"对号入座"。这是面向对象最优雅的特性之一。

```javascript
// 文件：code/ch05/05-inheritance.js（使用）
const zoo = [generic, dog, cat];
for (const a of zoo) {
  console.log(a.speak()); // 每个对象调用各自的 speak，各不相同
}
```

一个循环、一行 `a.speak()`，就让三种动物各自发声——不用 `if...else` 判断类型。这就是多态带来的简洁。

### instanceof：检查原型链

```javascript
// 文件：code/ch05/05-inheritance.js（使用）
console.log(dog instanceof Dog); // true
console.log(dog instanceof Animal); // true（Dog 继承自 Animal）
console.log(cat instanceof Dog); // false
console.log(Object.getPrototypeOf(Dog.prototype) === Animal.prototype); // true
```

`obj instanceof Class` 检查 `Class.prototype` **是否出现在 `obj` 的原型链上**。`dog` 的原型链是 `dog → Dog.prototype → Animal.prototype → Object.prototype → null`，所以对 `Dog` 和 `Animal` 都返回 `true`。最后一行直接验证了继承在底层就是**原型链的串联**——`extends` 干的事，就是把子类原型的原型接到父类原型上。

## 5.6 封装、多态与"组合优于继承"

到这里，面向对象的**三大特性**你已经全部见过了，我们总结一下：

- **封装（encapsulation）**：把数据和操作数据的方法打包在一起，并**隐藏内部细节**，只暴露必要接口。JS 里可用**私有字段 `#`**（首选）或**闭包**（第 4 章的 `makeCounter` 就是用闭包藏住了 `count`）实现。
- **继承（inheritance）**：子类复用父类的结构与行为（`extends` / `super`）。
- **多态（polymorphism）**：同一接口、不同实现，运行时按实际对象分派（上一节的 `speak`）。

### 继承并非万能：组合的思想

继承很好用，但**滥用继承会带来麻烦**。如果你设计一棵很深的继承树（A→B→C→D→……），一旦父类改动，所有子类可能受牵连；而现实世界的分类也常常是"横向"的能力，而非"纵向"的血缘。比如"鸭子会飞、会游、会走"，但"飞"这个能力，鸟、飞机、超人都有，硬塞进一棵继承树会很别扭。

于是有了一条广受推崇的设计原则：📌 **组合优于继承（composition over inheritance）**——与其"是一个（is-a）"的继承关系，不如"有一个能力（has-a）"的**拼装**。

```javascript
// 文件：code/ch05/06-composition.js
const canFly = (obj) => ({ fly: () => `${obj.name} 正在飞` });
const canSwim = (obj) => ({ swim: () => `${obj.name} 正在游` });
const canWalk = (obj) => ({ walk: () => `${obj.name} 正在走` });

function createDuck(name) {
  const duck = { name };
  return Object.assign(duck, canFly(duck), canSwim(duck), canWalk(duck));
}

function createFish(name) {
  const fish = { name };
  return Object.assign(fish, canSwim(fish));
}

const donald = createDuck("唐老鸭");
console.log(donald.fly()); // 唐老鸭 正在飞
console.log(donald.swim()); // 唐老鸭 正在游

const nemo = createFish("尼莫");
console.log(nemo.swim()); // 尼莫 正在游
```

**逐段拆解：**

- 每个"能力"（`canFly` / `canSwim` / `canWalk`）是一个小函数，接收对象、返回带某方法的对象。
- `Object.assign(目标, ...来源)` 把多个来源对象的属性**拷贝合并**到目标对象上。`createDuck` 给鸭子拼上"飞、游、走"三种能力。
- 鱼 `nemo` 只组合了 `canSwim`，所以它没有 `fly`——你**按需拼装**能力，不必背负庞大的继承树。

💡 **实践建议**：继承适合表达清晰、稳定的"是一个"关系（正方形**是一个**矩形）；当关系是"能做某事""有某功能"这类可自由搭配的能力时，优先用**组合**。二者并非对立，实际项目里常常混用。

## 常见坑与最佳实践

- ⚠️ **忘记 `new`（构造函数写法）**：`const u = User("A", 1)`（漏了 `new`）会让 `this` 指向全局或 `undefined`，悄悄出错。**改用 `class`**：不写 `new` 直接调用会当场报错，避免踩坑。

- ⚠️ **子类 constructor 里 `super` 之前用 `this`**：如 `this.x = 1; super();`，直接抛 `ReferenceError`。**永远先 `super(...)`，再碰 `this`**。

- ⚠️ **箭头函数当方法会丢 `this`**：类里如果把方法写成 `greet = () => {...}` 挂成字段，它会绑定实例的 `this`，看似方便，但**每个实例各存一份、不再共享原型上的方法**，也无法被子类用 `super` 调用。普通方法（`greet() {...}`）才挂在原型上、可被继承和重写。需要"回调里不丢 this"时再用箭头字段。

- ⚠️ **修改内置原型（monkey-patch）**：别去改 `Array.prototype`、`Object.prototype` 添加自定义方法，会污染所有对象、和未来的语言特性冲突、破坏 `for...in`。要扩展就用工具函数或子类。

- ⚠️ **`__proto__` 不是标准写法**：读写原型请用 `Object.getPrototypeOf` / `Object.setPrototypeOf`。而且**运行时改原型（`setPrototypeOf`）会严重拖慢性能**，尽量在创建时就定好原型。

- 📌 **优先私有字段 `#` 做封装**：需要真正隐藏的内部状态用 `#`；只读的对外暴露用 getter。别用"下划线前缀 `_balance`"假装私有——那只是约定，外部照样能改。

- 💡 **判断类型**：`instanceof` 检查原型链归属；`obj.constructor` 可查构造者；跨 iframe/realm 等场景 `instanceof` 可能失灵，那时改用鸭子类型（检查是否有某方法）或 `Symbol.hasInstance`。

- 💡 **不确定该不该继承时，先想"组合"**：能力可拼装的场景，组合往往更灵活、更易测试。

## 本章小项目：几何图形库

现在把本章所有知识——`class`、继承、`super`、私有字段、多态——揉进一个完整项目：**几何图形库**。

**需求**：定义抽象基类 `Shape`，派生 `Circle`（圆）、`Rectangle`（矩形）、`Square`（正方形，继承自 `Rectangle`）、`Triangle`（三角形）。每个图形实现 `area()`（面积）、`perimeter()`（周长）、`toString()`。最后用一个数组装下各种图形，遍历打印并求总面积——这一步会淋漓尽致地展示**多态**。

### 基类 Shape

```javascript
// 文件：code/ch05/shapes.js
class Shape {
  #name; // 私有：图形名字，外部只读

  constructor(name) {
    if (new.target === Shape) {
      throw new Error("Shape 是抽象基类，不能直接实例化，请使用其子类。");
    }
    this.#name = name;
  }

  get name() {
    return this.#name;
  }

  area() {
    throw new Error(`${this.name} 未实现 area() 方法`);
  }

  perimeter() {
    throw new Error(`${this.name} 未实现 perimeter() 方法`);
  }

  toString() {
    return `${this.name}｜面积=${this.area().toFixed(2)}｜周长=${this.perimeter().toFixed(2)}`;
  }
}
```

**逐段拆解：**

- `#name` 是私有字段，配合 `get name()` 对外只读，外部无法乱改图形名字——封装。
- **`new.target`** 是一个特殊值：用 `new` 调用时它指向"被 new 的那个类"。`new Shape()` 时 `new.target === Shape` 为真，我们**抛错**，阻止直接实例化基类。而 `new Circle()` 时 `new.target` 是 `Circle`，条件为假，正常放行。这样就模拟了"抽象类（abstract class）"。
- `area()` / `perimeter()` 在基类里**故意抛错**：基类不知道具体怎么算，逼子类去实现。这是"抽象方法"的模拟。
- `toString()` 是**通用逻辑**：它调用 `this.area()` 和 `this.perimeter()`，但**具体算哪种面积由运行时的子类决定**——多态的伏笔。`toFixed(2)` 保留两位小数。

### 圆 Circle

```javascript
// 文件：code/ch05/shapes.js（续）
class Circle extends Shape {
  #radius;

  constructor(radius) {
    super("圆"); // 先调父类构造，设置名字
    this.#radius = radius;
  }

  get radius() {
    return this.#radius;
  }

  area() {
    return Math.PI * this.#radius ** 2; // πr²
  }

  perimeter() {
    return 2 * Math.PI * this.#radius; // 2πr
  }
}
```

`Circle` 继承 `Shape`，`super("圆")` 把名字传给基类。它**重写**了 `area`（πr²）和 `perimeter`（2πr）。`**` 是幂运算符，`r ** 2` 即 r 的平方。`Math.PI` 是圆周率常量。

### 矩形 Rectangle 与正方形 Square

```javascript
// 文件：code/ch05/shapes.js（续）
class Rectangle extends Shape {
  #width;
  #height;

  constructor(width, height, name = "矩形") {
    super(name);
    this.#width = width;
    this.#height = height;
  }

  get width() { return this.#width; }
  get height() { return this.#height; }

  area() {
    return this.#width * this.#height;
  }

  perimeter() {
    return 2 * (this.#width + this.#height);
  }
}

class Square extends Rectangle {
  constructor(side) {
    super(side, side, "正方形"); // 宽高都传 side
  }

  get side() {
    return this.width; // 复用父类的 getter
  }
}
```

**逐段拆解：**

- `Rectangle` 的 constructor 多设计了一个 `name = "矩形"` 参数，默认叫"矩形"，但**允许子类改名**——这是为 `Square` 铺路。
- `Square extends Rectangle`：正方形**就是**宽高相等的矩形，用继承表达"is-a"关系非常贴切。`super(side, side, "正方形")` 把宽高都设成 `side`，并把名字改成"正方形"。
- `Square` **无需重写** `area` / `perimeter`——直接复用 `Rectangle` 的实现（边长平方、四倍边长），一行都不用写。`side` getter 复用了父类的 `width`。这就是继承的复用价值。

### 三角形 Triangle

```javascript
// 文件：code/ch05/shapes.js（续）
class Triangle extends Shape {
  #a;
  #b;
  #c;

  constructor(a, b, c) {
    super("三角形");
    if (a + b <= c || a + c <= b || b + c <= a) {
      throw new Error(`边长 ${a}, ${b}, ${c} 无法构成三角形`);
    }
    this.#a = a;
    this.#b = b;
    this.#c = c;
  }

  perimeter() {
    return this.#a + this.#b + this.#c;
  }

  area() {
    const s = this.perimeter() / 2; // 半周长
    return Math.sqrt(s * (s - this.#a) * (s - this.#b) * (s - this.#c));
  }
}
```

**逐段拆解：**

- 构造时**校验合法性**：任意两边之和必须大于第三边，否则抛错（构造函数里做参数校验是好习惯，第 6 章会专门讲错误处理）。
- `perimeter()` 就是三边相加。
- `area()` 用**海伦公式（Heron's formula）**：设半周长 `s = 周长 / 2`，面积 = √(s(s-a)(s-b)(s-c))。注意这里**复用了 `this.perimeter()`**，避免重复写三边相加——方法之间可以互相调用。

### 组装与多态遍历

```javascript
// 文件：code/ch05/shapes.js（续）
const shapes = [
  new Circle(5),
  new Rectangle(4, 6),
  new Square(3),
  new Triangle(3, 4, 5),
];

console.log("=== 图形清单 ===");
for (const shape of shapes) {
  console.log(shape.toString()); // 多态：每个图形算法不同
}

const totalArea = shapes.reduce((sum, shape) => sum + shape.area(), 0);
console.log("=== 汇总 ===");
console.log(`图形数量：${shapes.length}`);
console.log(`总面积：${totalArea.toFixed(2)}`);

const biggest = shapes.reduce((max, s) => (s.area() > max.area() ? s : max));
console.log(`面积最大的是：${biggest.name}（${biggest.area().toFixed(2)}）`);
```

**这段是全章的高潮：**

- `shapes` 数组里装着**四种不同类型**的对象，但它们都是 `Shape` 的子类，都有 `area()` / `perimeter()` / `toString()`。
- `for...of` 循环里只写了一句 `shape.toString()`，**却让每种图形按各自的公式计算**——圆用 πr²、矩形用宽×高、三角形用海伦公式。调用方**完全不需要知道**每个图形是什么类型，更没有 `if (是圆) ... else if (是矩形) ...`。这就是多态的威力：**统一的接口，各异的实现。**
- `reduce` 把每个图形的面积累加得到 `totalArea`（第 4 章学过），又用 `reduce` 挑出面积最大的图形。

## 小结与自测

本章我们揭开了 JavaScript 面向对象的全貌：

- **工厂函数**能批量造对象，但方法会重复创建，引出了**原型**。
- **原型（prototype）** 是对象的"备用对象"；读属性时沿**原型链**层层向上查找，直到 `Object.prototype → null`。`hasOwnProperty` 只看自己，`in` 连原型链看；写属性只写在自己身上（**遮蔽**）。
- **`new` 做四步**：建新对象 → 原型指向 `构造函数.prototype` → 以新对象为 `this` 执行构造 → 返回该对象。我们手写了 `myNew` 印证。
- **`class` 是原型的语法糖**：constructor、实例方法（挂在 prototype）、公有/私有字段（`#`）、`static` 静态成员、getter/setter，方法本质仍在 `prototype` 上。
- **继承**用 `extends` + `super`：子类 constructor 必须先 `super(...)` 再用 `this`；方法里 `super.方法()` 调父类实现；同名方法可**重写**。`instanceof` 沿原型链判断归属。
- **封装、继承、多态**是三大特性；**多态**让"统一接口、各异实现"成为可能，几何图形库淋漓尽致地展示了它。
- **组合优于继承**：能力可自由拼装时，用 `Object.assign` 组合往往比深继承更灵活。

（参考答案：1. 每个对象都复制一份相同方法，浪费内存；原型让所有实例共享同一份方法。2. 建新对象→原型指向构造函数的 prototype→以它为 this 执行构造→返回它；`Object.create` 完成前两步、`apply` 完成第三步的 this 绑定。3. 打印 `1 2 true`——`b.x=2` 在 b 自己身上建了 x 遮蔽了 a 的，a.x 不变，且 x 是 b 自己的属性。4. 方法在 `类.prototype` 上共享，公有字段在每个实例上各一份；`#x` 只能在类内访问，外部访问会语法报错。5. 因为父类构造未跑完 `this` 尚未初始化；constructor 里 `super()` 是调父类构造函数，方法里 `super.m()` 是调父类的方法 m。6. 同一接口不同实现、运行时按实际对象分派；`shape.toString()` 对每种图形调用各自的 `area`，无需类型判断。7. 因为 `Animal.prototype` 在 dog 的原型链上；`instanceof` 检查右侧的 `prototype` 是否出现在左侧对象的原型链中。）
