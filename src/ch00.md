## 0.1 JavaScript 运行在哪里？

很多人以为"JavaScript 是浏览器里的语言"。这句话在 2009 年之前是对的，但今天只对了一半。

JavaScript 是一门**语言**，它需要一个**引擎（engine）**来执行。最有名的引擎是 Google 的 **V8**（Chrome 浏览器内置）。2009 年，有人把 V8 引擎从浏览器里"抠"出来，装上文件读写、网络等能力，做成了一个可以在**操作系统上直接运行 JS**的程序——这就是 **Node.js**。

于是今天 JavaScript 有两大运行环境：

| 环境 | 典型场景 | 能做的事 | 不能做的事 |
| --- | --- | --- | --- |
| **浏览器** | 网页交互 | 操作网页（DOM）、发网络请求 | 直接读写你电脑的文件 |
| **Node.js** | 后端服务、命令行工具、构建脚本 | 读写文件、开网络服务、访问数据库 | 操作网页（没有 DOM） |

本书大部分示例用 **Node.js** 运行——它更接近"通用编程语言"的体验，不需要打开浏览器就能跑代码，也是我们后面做后端开发的基础。只有讲 DOM 的第 10 章会回到浏览器。

> 📌 记住这个心智模型：**语言是 JavaScript，引擎是 V8，运行环境是浏览器或 Node.js。** 后面遇到"这个函数为什么在 Node 里有、浏览器里没有"之类的问题，答案往往就在这张表里。

## 0.2 安装 Node.js

打开 [nodejs.org](https://nodejs.org/)，下载 **LTS（长期支持）** 版本安装即可。LTS 是稳定版，适合学习和生产。

安装完成后，打开终端（macOS 用"终端 Terminal"，Windows 用"PowerShell"），输入下面两条命令验证：

```bash
node --version
npm --version
```

如果分别打印出类似 `v20.x.x` 和 `10.x.x` 的版本号，说明安装成功。

这里出现了两个命令：

- **`node`**：Node.js 本体，用来运行 JS 代码。
- **`npm`**（Node Package Manager）：随 Node 一起装好的**包管理器**，用来下载别人写好的代码库（第 9 章详解）。

> 💡 **进阶（可选）**：如果你以后要在多个 Node 版本间切换，可以用 [nvm](https://github.com/nvm-sh/nvm)（macOS/Linux）或 [nvm-windows](https://github.com/coreybutler/nvm-windows) 管理版本。现在不必纠结，装个 LTS 就够学完全书。

## 0.4 运行 JS 代码的三种方式

### 方式一：REPL（交互式，随手试）

在终端直接输入 `node` 回车，你会进入一个可以逐行输入、立刻得到结果的环境，这叫 **REPL**（Read-Eval-Print-Loop，读取-求值-打印-循环）：

```text
$ node
Welcome to Node.js v20.x.x.
> 1 + 1
2
> const name = "world"
undefined
> `hello, ${name}`
'hello, world'
> .exit
```

REPL 非常适合快速验证一个语法、试一个小表达式。想退出输入 `.exit` 或按两次 `Ctrl+C`。

> ⚠️ 注意上面 `const name = "world"` 那行返回了 `undefined`——这不是报错。声明变量这个"动作"本身没有返回值，REPL 就显示 `undefined`。下一行用到 `name` 时，它确实是 `"world"`。

### 方式二：运行 `.js` 文件（最常用）

真正写程序时，我们把代码存进文件。新建一个文件 `hello.js`：

```javascript
// 文件：code/ch00/hello.js
console.log("你好，JavaScript！");
console.log("1 到 5 的和是：", 1 + 2 + 3 + 4 + 5);
```

`console.log(...)` 是最常用的一句代码，作用是把括号里的内容**打印到终端**。它是你调试程序、观察结果的头号工具。

在终端进入文件所在目录，运行：

```bash
node hello.js
```

你会看到：

```text
你好，JavaScript！
1 到 5 的和是： 15
```

恭喜，这就是一个完整的程序在运行了。**本书绝大多数示例都用这种方式运行。**

### 方式三：浏览器控制台

在任意网页按 `F12`（或右键"检查"）打开开发者工具，切到 **Console（控制台）** 标签，也能直接输入并运行 JS。这在调试网页时很有用，我们到第 10 章讲 DOM 时会用到。

## 0.5 提前认识：运行 TypeScript

TypeScript 的代码存在 `.ts` 文件里，但 **Node.js 不能直接运行 `.ts` 文件**（它只认 JavaScript）。所以 TS 需要先"翻译"成 JS，这个过程叫**编译（compile）**。有两种常见做法，第 13 章会详细讲，这里先混个眼熟：

```bash
# 做法一：先编译成 .js，再用 node 运行
npx tsc hello.ts     # 生成 hello.js
node hello.js

# 做法二：用 tsx 一步到位（内部帮你即时编译再运行）
npx tsx hello.ts
```

`npx` 是随 npm 一起来的命令，作用是"临时下载并运行一个工具"。本书 `code/` 目录已经配好了 `tsx`，你在 `code/` 下 `npm install` 之后，就能用 `npx tsx 某文件.ts` 直接跑 TS 示例。

> 📌 现在**不需要**真的去装 TypeScript，这一节只是让你知道"TS 要先编译"这件事。等学到第 13 章我们再正式动手。

## 0.6 建立你的代码目录

学习本书时，建议你自己建一个练习目录，比如 `my-practice/`，每章建一个子文件夹，把示例亲手敲进去。这样你的练习和本书自带的 `code/`（标准答案）互不干扰。


第一次使用前，在 `code/` 目录执行一次 `npm install`，把 `tsx`、`vitest` 等工具装好（第 9 章会讲清楚这一步在做什么）：

```bash
cd code
```

## 本章小项目：环境自检脚本

**需求**：写一个脚本 `doctor.js`，运行后打印当前环境信息，确认一切就绪。这既能验证你的环境，又能让你熟悉 `node` 运行文件的流程，还顺便认识几个 Node 内置对象。

**目标输出**（数字会因你的机器而不同）：

```text
==============================
  环境自检 Environment Doctor
==============================
Node.js 版本 : v20.11.0
操作系统     : darwin (arm64)
当前工作目录 : /Users/you/master_book/code/ch00
运行时间     : 2026-07-02 10:30:00
✅ 恭喜，你的环境已经可以开始学习了！
```

### 完整实现

```javascript
// 文件：code/ch00/doctor.js
// 运行方式：node doctor.js

// process 是 Node.js 提供的全局对象，代表"当前运行的这个进程"，
// 里面藏着版本、平台、工作目录等信息。
const nodeVersion = process.version; // 例如 "v20.11.0"
const platform = process.platform; // "darwin"(mac) / "win32" / "linux"
const arch = process.arch; // CPU 架构，如 "arm64" / "x64"
const cwd = process.cwd(); // current working directory，当前所在目录

// 生成一个"年-月-日 时:分:秒"格式的时间字符串。
// new Date() 拿到当前时间，后面这些方法把它拆成各部分。
const now = new Date();
const pad = (n) => String(n).padStart(2, "0"); // 个位数前面补 0，如 5 -> "05"
const timestamp =
  `${now.getFullYear()}-${pad(now.getMonth() + 1)}-${pad(now.getDate())} ` +
  `${pad(now.getHours())}:${pad(now.getMinutes())}:${pad(now.getSeconds())}`;

// 开始打印报告。
console.log("==============================");
console.log("  环境自检 Environment Doctor");
console.log("==============================");
console.log("Node.js 版本 :", nodeVersion);
console.log("操作系统     :", `${platform} (${arch})`);
console.log("当前工作目录 :", cwd);
console.log("运行时间     :", timestamp);

// 简单校验：主版本号是否 >= 18（本书示例的推荐下限）。
// nodeVersion 形如 "v20.11.0"，slice(1) 去掉开头的 "v"，
// split(".") 按点切成 ["20","11","0"]，取第 0 个并转成数字。
const majorVersion = Number(nodeVersion.slice(1).split(".")[0]);

if (majorVersion >= 18) {
  console.log("✅ 恭喜，你的环境已经可以开始学习了！");
} else {
  console.log("⚠️ 你的 Node 版本偏低，建议升级到 LTS（18 及以上）。");
}
```

### 逐段讲解

1. **`process` 对象**：这是 Node.js 的全局对象之一，不用 `import` 就能直接用。它代表"当前这个正在运行的 Node 进程"。`process.version`、`process.platform` 是属性（直接取值），`process.cwd()` 是方法（要加括号调用）——**属性像名词、方法像动词**，这个区分以后会反复用到。

2. **`new Date()`**：创建一个表示"此刻"的时间对象。`getMonth()` 有个著名的坑：它返回的月份从 **0** 开始（0 表示一月），所以要 `+ 1`。这类"反直觉"的设计，正是第 3 章讲对象和第 11 章讲日期时会提醒你的地方。

3. **`pad` 函数与 `padStart`**：`padStart(2, "0")` 的意思是"把字符串补到至少 2 位，不够就在前面填 `0`"。于是 `5` 变成 `"05"`，时间显示更整齐。这里我们第一次用到**箭头函数** `(n) => ...`，第 2 章会正式讲函数。

4. **模板字符串**：用反引号 `` ` `` 包起来、内部用 `${...}` 嵌入变量的写法，叫**模板字符串（template literal）**，比用 `+` 拼接清爽得多。第 1 章详解。

5. **`if / else` 判断**：根据主版本号给出不同提示。`Number(...)` 把字符串 `"20"` 转成数字 `20`，才能和 `18` 做大小比较——**类型转换**是第 1 章的重点。

### 运行它

```bash
cd code/ch00
node doctor.js
```

看到最后那行 `✅` 就说明你的环境完全就绪了。

### 扩展练习

1. 让脚本额外打印 `process.env.HOME`（Mac/Linux）或 `process.env.USERPROFILE`（Windows），显示你的用户主目录。想想 `process.env` 是什么？（提示：环境变量，第 24 章会重点用到。）
2. 把版本下限从 18 改成一个很高的值（比如 999），运行看看会走到哪个分支、打印什么。体会 `if/else` 的分支逻辑。
3. 试着把 `doctor.js` 改写成 `doctor.ts`（改扩展名、内容不变），然后用 `npx tsx doctor.ts` 运行。你会发现——它照样能跑！因为**所有合法的 JS 都是合法的 TS**。这就是"TS 是 JS 超集"的第一手体验。

## 小结与自测

**本章重点回顾：**

- JavaScript 是语言，V8 是引擎，浏览器和 Node.js 是两大运行环境。
- Node.js 让 JS 能脱离浏览器、直接在操作系统上运行，是本书的主力环境。
- `node 文件.js` 是最常用的运行方式；`node` 直接回车进入 REPL 适合随手试。
- TypeScript 要先编译成 JS 才能运行，本书用 `npx tsx` 一步执行。
- `console.log` 是你最好的朋友；`process` 对象藏着运行环境的各种信息。

**自测题：**

1. 为什么在浏览器里能操作网页、在 Node.js 里却不行？反过来 Node 能读写文件、浏览器却不行，原因是什么？
2. `node` 直接回车和 `node 文件.js` 有什么区别？各自适合什么场景？
3. 运行 `.ts` 文件为什么不能直接 `node xxx.ts`？中间少了哪一步？
4. `process.cwd()` 后面为什么有括号，而 `process.version` 后面没有？
