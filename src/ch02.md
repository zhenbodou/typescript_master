# 第 2 章 · 控制流与函数基础

上一章我们学会了用变量装数据、用运算符算数据。但一个只会"从上往下一条道走到黑"的程序是没什么用的——它不会**根据情况做选择**，也不会**重复做事**，更不会把一段逻辑**打包起来反复调用**。

这一章，我们就来补齐这三块：

- **控制流（control flow）**：让程序学会"如果……就……"（条件）和"重复做直到……"（循环）。
- **函数（function）**：把一段逻辑装进一个"盒子"，取个名字，随时调用。

学完这一章，你就能写出真正"有脑子"的程序了。本章末尾我们会做一个能在终端和你来回互动的**猜数字游戏**。

## 本章目标

- 用 `if / else if / else` 和 `switch` 让程序做条件判断，并绕开 `switch` 的经典陷阱。

- 掌握五种循环：`for`、`while`、`do...while`、`for...of`、`for...in`，并分清 `for...of` 和 `for...in` 的区别。

- 用阶乘入门**递归（recursion）**。

## 2.1 条件判断：if / else if / else

程序做决定，最基础的工具就是 `if`。它的意思很直白：**如果括号里的条件为真，就执行大括号里的代码。**

```javascript
// 文件：code/ch02/01-if.js
const score = 76;

if (score >= 90) {
  console.log("优秀");
} else if (score >= 80) {
  console.log("良好");
} else if (score >= 60) {
  console.log("及格");
} else {
  console.log("不及格");
}
```

运行 `node 01-if.js`，因为 `score` 是 76，会打印"及格"。

### 逐段讲解

1. **执行顺序是从上往下**：JavaScript 先看 `score >= 90`，为假；再看 `score >= 80`，还是假；再看 `score >= 60`，为真——**立刻执行这个分支，然后整个 if 结构就结束了**，后面的 `else` 根本不会再看。这一点很重要：**只会命中第一个为真的分支。**

2. **顺序会影响结果**：正因为"命中即停"，条件的排列顺序很关键。如果你把 `score >= 60` 写在最前面，那么 95 分也会先命中它、打印"及格"。所以这种"分档"判断，要**从最严格的条件往最宽松的排**。

3. **`else if` 可以有很多个，`else` 最多一个且可省略**。只有一个条件时，`else` 完全可以不写：

```javascript
const isRaining = true;
if (isRaining) {
  console.log("记得带伞");
}
```

> 📌 `if (...)` 括号里的东西最终会被当成布尔值来看。回忆第 1 章讲的**真值 / 假值（truthy / falsy）**：`0`、`""`、`null`、`undefined`、`NaN`、`false` 是假值，其余都是真值。所以 `if (score)` 在 `score` 为 0 时不会进去——这常常是 bug 的来源。

### ⚠️ 头号陷阱：把 `=` 当成 `===`

初学者最容易犯的错，是在条件里把"赋值 `=`"写成了"比较 `===`"：

```javascript
let x = 0;
if ((x = 1)) {
  console.log("这行一定会执行，因为 x 被赋成了 1（真值）");
}
console.log("现在 x 的值被改成了：", x); // 1
```

`x = 1` 不是"判断 x 等不等于 1"，而是"把 1 赋给 x"，这个表达式的结果就是 1（真值），于是 `if` 永远成立，顺便还偷偷改了 `x`。判断相等永远用 `===`（第 1 章讲过，优先用严格相等，别用 `==`）。

## 2.2 switch：多分支的另一种写法

当你要拿**同一个值**去和**一串固定选项**逐个比对时，`switch` 有时比一长串 `else if` 更清爽：

```javascript
// 文件：code/ch02/02-switch.js
const day = 3;

switch (day) {
  case 1:
    console.log("星期一");
    break;
  case 2:
    console.log("星期二");
    break;
  case 3:
    console.log("星期三");
    break;
  default:
    console.log("其他日子");
}
```

### 逐段讲解

1. `switch (day)` 取出要比对的值 `day`。
2. 每个 `case 3:` 表示"如果 `day` 严格等于 3"。注意是**严格相等 `===`**——所以 `case "3"`（字符串）不会匹配数字 `3`。
3. `default` 相当于 `else`，所有 `case` 都没命中时执行，可以省略（但建议留着，处理"意外情况"）。
4. 每个分支末尾的 **`break` 至关重要**：它表示"这个分支处理完了，跳出 switch"。

### ⚠️ fall-through：忘记 break 会"穿透"

如果某个 `case` 里忘了写 `break`，程序**不会停下来**，而是继续执行下一个 `case` 的代码，直到遇到 `break` 或 switch 结束。这叫 **fall-through（穿透）**：

```javascript
const color = "red";
switch (color) {
  case "red":
    console.log("红色"); // 命中这里
  // 没有 break，继续往下穿透！
  case "green":
    console.log("绿色");
  case "blue":
    console.log("蓝色");
    break;
  default:
    console.log("未知颜色");
}
```

`color` 是 `"red"`，命中第一个 `case` 后由于没有 `break`，会一路打印"红色""绿色""蓝色"三行才停。这几乎总是 bug。**养成习惯：每个 case 都写 break。**

### 💡 有意利用 fall-through：合并 case

fall-through 唯一正当的用法，是让**多个 case 共享同一段逻辑**——故意不写 `break`，让它们落到同一处：

```javascript
const month = 4;
switch (month) {
  case 3:
  case 4:
  case 5:
    console.log("春天");
    break;
  case 6:
  case 7:
  case 8:
    console.log("夏天");
    break;
  default:
    console.log("其他季节");
}
```

`month` 是 3、4、5 中任何一个，都会落到同一句 `console.log("春天")`。这种写法是可读的，因为空的 `case` 一眼就能看出"是在合并"。

> 📌 什么时候用 `switch`、什么时候用 `if`？规则很简单：**拿一个值和一堆固定常量比对，用 `switch`；条件是范围判断（如 `>= 90`）或涉及多个变量，用 `if / else if`。** 上面的分数分档就更适合 `if`。

## 2.3 三元运算符：一行搞定的二选一

第 1 章我们见过三元运算符（ternary operator），这里复习并说清它的定位。它的形式是：

```
条件 ? 为真时的值 : 为假时的值
```

关键点：它是一个**表达式（expression）**，会算出一个**值**，所以可以直接赋给变量：

```javascript
// 文件：code/ch02/03-ternary.js
const age = 20;
const label = age >= 18 ? "成年人" : "未成年人";
console.log(label); // 成年人
```

这等价于下面这段更啰嗦的 `if/else`，但只用一行：

```javascript
let label2;
if (age >= 18) {
  label2 = "成年人";
} else {
  label2 = "未成年人";
}
```

### 什么时候用三元

**当你要根据条件在两个"值"之间二选一时**，三元最合适，尤其是嵌在模板字符串里：

```javascript
const n = 7;
console.log(`${n} 是${n % 2 === 0 ? "偶数" : "奇数"}`); // 7 是奇数
```

### ⚠️ 别嵌套太深

三元可以嵌套，但嵌套两层以上就非常难读了：

```javascript
const s = 85;
const grade = s >= 90 ? "A" : s >= 80 ? "B" : s >= 60 ? "C" : "D";
```

这行还能勉强看懂，但再复杂就该老老实实换回 `if / else if`。**代码是给人读的**——能用三元让一行更清晰就用，会让一行变成天书就别用。

## 2.4 循环：让程序重复做事

如果没有循环，"打印 1 到 100"你得写 100 行 `console.log`。循环让你把"重复的事"交给程序。JavaScript 有好几种循环，我们一个个来。

### 2.4.1 for 循环

`for` 是最经典的循环，适合"我知道要循环多少次"的场景。它的括号里有**三段**，用分号隔开：

```javascript
// 文件：code/ch02/04-loops.js
for (let i = 1; i <= 5; i++) {
  console.log("第", i, "次");
}
```

三段分别是：

1. **初始化 `let i = 1`**：循环开始前执行一次，通常用来定义计数器。
2. **继续条件 `i <= 5`**：每轮循环**开始前**检查，为真才执行循环体，为假就结束整个循环。
3. **更新 `i++`**：每轮循环体执行**完之后**执行，通常用来让计数器前进。

所以这段代码的执行流程是：`i=1` 检查 `1<=5`✓ 打印 → `i++` 变 2 → 检查 `2<=5`✓ 打印 → …… → `i` 变 6 → 检查 `6<=5`✗ 结束。一共打印 5 行。

> 💡 `i++` 是 `i = i + 1` 的简写（第 1 章讲过自增运算符）。计数器习惯用 `i`、`j`、`k` 这些名字，是几十年的编程传统。

### 2.4.2 while 循环

`while` 更朴素：**只要条件为真，就一直重复。** 适合"不知道要循环几次，但知道什么时候该停"的场景：

```javascript
let count = 3;
while (count > 0) {
  console.log("倒计时", count);
  count--; // ⚠️ 忘了这句会变成死循环！
}
console.log("发射！");
```

这里 `count` 从 3 开始，每轮减 1，减到 0 时 `count > 0` 为假、循环结束。

> ⚠️ **死循环（infinite loop）**：`while` 的循环体里**必须有让条件最终变假的语句**（这里是 `count--`）。如果忘了写，`count` 永远是 3，程序会永远转下去、卡死。真卡住了，在终端按 `Ctrl + C` 强制结束。

### 2.4.3 do...while 循环

`do...while` 和 `while`几乎一样，唯一区别是：**它先执行一次循环体，再检查条件。** 所以循环体**至少会跑一次**，哪怕条件一开始就是假的：

```javascript
let m = 100;
do {
  console.log("这行至少执行一次，即使条件一开始就为假");
} while (m < 10);
```

`m` 是 100，`m < 10` 一开始就为假，但因为是"先做后判断"，那行 `console.log` 仍然执行了一次。这种"至少执行一次"的特性，很适合"先让用户输入一次，再判断要不要继续"——我们的猜数字游戏就是这个模式。

### 2.4.4 break 与 continue

在循环里，你可以用两个关键字精确控制流程：

- **`break`**：立刻**跳出整个循环**，不再继续。
- **`continue`**：**跳过本轮剩下的代码**，直接进入下一轮。

```javascript
// break：找到目标就停
for (let i = 1; i <= 10; i++) {
  if (i === 4) {
    break; // i 到 4 就跳出，后面的 5~10 都不跑了
  }
  console.log(i); // 打印 1 2 3
}

// continue：跳过不想处理的
for (let i = 1; i <= 6; i++) {
  if (i % 2 === 0) {
    continue; // 是偶数就跳过，不执行下面的 console.log
  }
  console.log("奇数：", i); // 打印 1 3 5
}
```

`break` 好比"任务完成，提前收工"；`continue` 好比"这一件我不管，看下一件"。这两个在真实代码里极其常用。

## 2.5 for...of 与 for...in：遍历的两种姿势

前面的 `for` 靠计数器循环。但更多时候我们是想**逐个处理一个集合里的元素**（比如数组）。这时有两个更方便的循环：`for...of` 和 `for...in`。它们长得像，用途却完全不同，是初学者最容易搞混的地方之一。

一句话记住：**`for...of` 拿"值"，`for...in` 拿"键"。**

```javascript
// 文件：code/ch02/05-for-of-in.js
// for...of：遍历数组的"值"
const fruits = ["苹果", "香蕉", "橙子"];
for (const fruit of fruits) {
  console.log(fruit); // 苹果 / 香蕉 / 橙子
}
```

`for...of` 每轮把数组里的**一个值**赋给 `fruit`。这正是遍历数组最自然的写法（数组细节第 3 章讲）。

`for...in` 则遍历一个对象的**键（key）**。它最正当的用途是遍历对象的属性名：

```javascript
const person = { name: "小明", age: 18, city: "北京" };
for (const key in person) {
  console.log(key, ":", person[key]); // name : 小明 ……
}
```

这里 `key` 依次是 `"name"`、`"age"`、`"city"`，再用 `person[key]` 取出对应的值（对象与这种"方括号取值"第 3 章详解）。

### ⚠️ 陷阱：不要用 for...in 遍历数组

很多人想遍历数组时随手写了 `for...in`，结果掉坑：

```javascript
const arr = [10, 20, 30];
arr.extra = "我是额外属性"; // 给数组挂了个属性
for (const key in arr) {
  console.log("key =", key, typeof key);
}
// 输出：
// key = 0 string
// key = 1 string
// key = 2 string
// key = extra string   ← 连额外属性都遍历出来了！
```

问题有三个：

1. **它给你的是下标"键"而不是值**，而且是**字符串** `"0"` 而非数字 `0`（注意 `typeof key` 是 `string`）。
2. **顺序不保证**（虽然数组下标通常按序，但规范上不承诺）。
3. **会把额外挂在数组上的属性也遍历出来**（上面的 `extra`）。

所以规则很硬：**数组用 `for...of`（或后面章节的 `forEach`、`map`），对象用 `for...in`。** 如果确实要在遍历数组时拿到下标，用 `for...of` 配合 `entries()`（第 3 章会讲）或经典的三段式 `for`。

> 💡 记忆口诀：**of → 值（value），in → 索引/键（index/key）。** 读起来也顺："for each fruit **of** fruits"（遍历水果们里的每个水果）。

## 2.6 函数：把逻辑装进盒子

到目前为止，我们的代码都是"一次性"的。**函数（function）** 让你把一段逻辑**打包、命名、反复调用**，还能接收不同的输入、给出不同的输出。它是组织代码最核心的工具。

函数有三种写法，先看第一种。

### 2.6.1 函数声明（function declaration）

```javascript
// 文件：code/ch02/06-functions.js
function add(a, b) {
  return a + b;
}
console.log("add(2, 3) =", add(2, 3)); // 5
```

拆解：

- **`function`** 是关键字，表示"我要定义一个函数"。
- **`add`** 是函数名，以后靠它来调用。
- **`(a, b)`** 是**形参（parameter）**——函数"约定要接收的输入"的占位名字。
- **`{ ... }`** 里是**函数体**，函数被调用时执行的代码。
- **`return a + b`** 把计算结果**交回给调用者**（下面细讲）。

`add(2, 3)` 这一步叫**调用（call / invoke）**。这里的 `2` 和 `3` 是**实参（argument）**——调用时真正传进去的值。调用时，`a` 拿到 2、`b` 拿到 3，算出 5 并 `return` 回来，于是 `add(2, 3)` 整体"变成"了 5。

> 📌 分清两个词：**形参（parameter）是定义函数时的占位名**（`a`、`b`）；**实参（argument）是调用时传的真实值**（`2`、`3`）。

### 2.6.2 函数表达式（function expression）

函数在 JavaScript 里也是一种"值"，因此可以像数字、字符串那样**赋给变量**：

```javascript
const multiply = function (a, b) {
  return a * b;
};
console.log("multiply(2, 3) =", multiply(2, 3)); // 6
```

这里等号右边是一个"没有名字的函数"（匿名函数），把它赋给变量 `multiply`，之后就用 `multiply(...)` 调用。这叫**函数表达式**。它和函数声明的功能几乎一样，但在"提升"上有区别（见 2.8）。

### 2.6.3 箭头函数（arrow function）

**箭头函数** 是 ES6 引入的更简洁写法，用 `=>` 代替 `function`：

```javascript
const square = (n) => n * n;
console.log("square(5) =", square(5)); // 25
```

箭头函数有两个"简写福利"：

1. **当函数体只有一个表达式时**，可以省略大括号和 `return`——那个表达式的值自动成为返回值。上面 `n * n` 就是被自动返回的。
2. 只有一个参数时，参数外的括号也能省（`n => n * n`），但为统一风格本书保留括号。

如果函数体是多行，就必须写大括号，并且要**显式 `return`**：

```javascript
const describe = (name, age) => {
  const stage = age >= 18 ? "成年" : "未成年";
  return `${name} 已${stage}`;
};
console.log(describe("小红", 16)); // 小红 已未成年
```

> 📌 箭头函数不只是"写得短"。它在 `this` 的处理上和普通函数有本质区别，还没有自己的 `arguments`。这些**留到第 4 章**专门讲 `this` 和闭包时展开。现在你只需把它当作"更简洁的函数写法"。

## 2.7 return 与返回值

`return` 做两件事：**① 把一个值交回给调用者；② 立刻结束函数的执行**（`return` 后面的代码不再运行）。

一个函数如果**没有 `return`**（或者 `return` 后面不带值），它的返回值就是 `undefined`：

```javascript
function sayHi(name) {
  console.log("你好，", name);
  // 没有 return
}
const result = sayHi("阿强");
console.log("sayHi 的返回值是：", result); // undefined
```

`sayHi` 只负责打印，没有把任何值 `return` 出来，所以 `result` 是 `undefined`。

> 📌 分清两件事：**`console.log` 是"打印给人看"，`return` 是"把值交给程序继续用"。** 初学者常混淆——一个函数在屏幕上打印了内容，不代表它"返回"了那个内容。要想让别的代码用到结果，必须 `return`。

> 💡 `return` 的"立刻结束"特性可以用来提前退出，让代码更扁平。比如参数不合法时 `return` 掉，避免层层嵌套的 `if`——这叫"卫语句（guard clause）"，第 6 章会再见到。

## 2.8 函数提升（hoisting）

看这段能正常运行的"怪代码"——我们在函数**定义之前**就调用了它：

```javascript
console.log("提前调用 declared()：", declared()); // 居然能跑！
function declared() {
  return "我是函数声明，可以被提前调用";
}
```

之所以能跑，是因为 JavaScript 在真正执行代码之前，会先"扫描"一遍，把所有**函数声明**整体"提"到它所在作用域的顶部——这叫 **函数提升（hoisting）**。所以在你看来 `declared()` 写在下面，但对引擎来说它"早就存在了"。

但**函数表达式和箭头函数不会被这样提升**。它们赋值给的变量（`const`/`let`）在赋值那行之前是"还没准备好"的：

```javascript
// console.log(expr()); // 若解开注释：报错 Cannot access 'expr' before initialization
const expr = () => "我是函数表达式，不能被提前调用";
console.log("正常调用 expr()：", expr()); // 这样才对
```

> 📌 结论：**函数声明会提升（可以先用后定义），函数表达式/箭头函数不会（必须先定义后用）。** 为了避免混乱，本书的建议是：**无论哪种，都养成"先定义、后使用"的习惯**，别去依赖提升这个"魔法"。

## 2.9 参数进阶：默认参数、剩余参数、arguments

### 默认参数（default parameters）

给形参一个默认值，调用时不传（或传 `undefined`）就用默认值：

```javascript
// 文件：code/ch02/07-params.js
function greet(name, greeting = "你好") {
  return `${greeting}，${name}！`;
}
console.log(greet("小明")); // 你好，小明！
console.log(greet("小红", "早上好")); // 早上好，小红！
```

第一次调用没传 `greeting`，于是用默认的 `"你好"`；第二次传了 `"早上好"`，就覆盖默认值。默认参数让函数更灵活，也省去了在函数体里手动判断"没传就用某值"的麻烦。

### 剩余参数（rest parameters）

有时你不知道会传进来几个参数。用 `...` 把"剩下的所有实参"打包成一个**真正的数组**：

```javascript
function sum(...nums) {
  let total = 0;
  for (const n of nums) {
    total += n;
  }
  return total;
}
console.log(sum()); // 0
console.log(sum(1, 2, 3)); // 6
console.log(sum(10, 20, 30, 40)); // 100
```

`...nums` 把所有传进来的数字收进数组 `nums`，然后我们用 `for...of` 累加。不管传几个都行。

前面可以有固定参数，但 **rest 参数必须放在最后**：

```javascript
function joinWith(separator, ...words) {
  return words.join(separator); // 用分隔符把词连起来
}
console.log(joinWith("-", "a", "b", "c")); // a-b-c
```

`separator` 吃掉第一个实参，剩下的 `"a"`、`"b"`、`"c"` 进 `words`。

### arguments（了解即可）

在**普通函数**（非箭头函数）里，还有一个自带的、名叫 `arguments` 的"类数组对象"，装着所有实参：

```javascript
function oldStyle() {
  console.log("arguments 长度：", arguments.length); // 3
  console.log("第一个实参：", arguments[0]); // x
}
oldStyle("x", "y", "z");
```

`arguments` 是老式写法，它不是真正的数组（不能直接用数组方法），而且箭头函数里根本没有它。**现代代码优先用剩余参数 `...args`**，这里介绍它只是为了让你日后读到老代码时不迷惑。

## 2.10 作用域基础：变量能被谁看见

**作用域（scope）** 决定了"一个变量在哪些地方能被访问到"。理解它，你才能明白很多"为什么这里读不到那个变量"的报错。这一节先打基础，第 4 章会深入到闭包。

### 函数作用域

函数内部声明的变量，**外面看不见**：

```javascript
// 文件：code/ch02/08-scope.js
function demo() {
  const inside = "我只在 demo 内部可见";
  console.log(inside);
}
demo();
// console.log(inside); // 解开会报错：inside is not defined
```

`inside` 是 `demo` 的"私有变量"，函数一执行完它就被销毁，外部无从访问。这其实是好事——它让函数成为一个封闭的"盒子"，内部变量不会污染外面。

### 块级作用域（let / const）

一对大括号 `{ }` 就构成一个**块（block）**。用 `let`/`const` 声明的变量，**只在所在的块里有效**：

```javascript
{
  let blockVar = "我在块里";
  console.log(blockVar);
}
// console.log(blockVar); // 解开会报错
```

`for` 循环里 `let i` 声明的 `i` 同样是块级的，循环一结束外面就访问不到 `i` 了。

### ⚠️ var 没有块级作用域

第 1 章说过"别用 `var`"，这里给你看它的一个坑。`var` **只认函数作用域，不认块**，会"泄漏"出块外：

```javascript
{
  var leaked = "我用 var 声明，居然泄漏到了块外面";
}
console.log(leaked); // 能打印出来！这就是 var 的坑
```

换成 `let` 就会正确报错。所以再强调一次：**一律用 `const`，需要重新赋值时才用 `let`，永远别用 `var`。**

### 内层能读外层：为闭包埋伏笔

作用域是**嵌套**的：内层代码能读到外层的变量，反过来不行。

```javascript
const outer = "外层变量";
function reader() {
  console.log("在函数里读到：", outer); // 能读到外层的 outer
}
reader();
```

函数 `reader` 内部没有 `outer`，于是往外层找，找到了。这个"函数能记住并访问它定义时所在的外层变量"的能力，正是第 4 章**闭包（closure）** 的核心，先在这里混个脸熟。

## 2.11 递归入门：函数调用自己

**递归（recursion）** 是指一个函数在自己的函数体里**调用它自己**。听起来有点绕，但很多问题用递归描述特别自然。

经典例子是**阶乘**：`n! = n × (n-1) × … × 1`，并规定 `0! = 1`。注意到 `n! = n × (n-1)!`——问题里套着一个"更小的同类问题"，这正是递归的信号：

```javascript
// 文件：code/ch02/09-recursion.js
function factorial(n) {
  // 终止条件（base case）：递归必须有出口，否则无限递归
  if (n <= 1) {
    return 1;
  }
  // 递归步骤：把 factorial(n) 拆成 n * factorial(n-1)
  return n * factorial(n - 1);
}

console.log("5! =", factorial(5)); // 120
console.log("10! =", factorial(10)); // 3628800
```

### 递归的两个必备零件

1. **终止条件（base case）**：一个"不再往下调用、直接返回结果"的出口。这里是 `n <= 1` 时返回 1。**没有终止条件，函数会无限调用自己，最终报"栈溢出（stack overflow）"错误。** 这和 `while` 忘了改条件变成死循环是一个道理。
2. **递归步骤**：把大问题化成一个**更接近终止条件的小问题**。这里 `factorial(n)` 依赖 `factorial(n - 1)`，参数每次减 1，早晚会到 1。

把 `factorial(3)` 的执行过程"展开"看，就很清楚了：

```
factorial(3)
  = 3 * factorial(2)
  = 3 * (2 * factorial(1))
  = 3 * (2 * 1)      ← factorial(1) 命中终止条件，返回 1
  = 3 * 2
  = 6
```

先"层层展开"直到触底，再"层层返回"把结果乘回来。

> 💡 **递归和循环常常可以互相改写。** 阶乘用普通 `for` 循环也能算：

```javascript
function factorialLoop(n) {
  let result = 1;
  for (let i = 2; i <= n; i++) {
    result *= i;
  }
  return result;
}
console.log(factorialLoop(5)); // 120
```

初学阶段，能用循环解决的就用循环（更省内存、更好懂）；等遇到树形结构、目录遍历这类"天然嵌套"的问题时，递归的威力才真正显现。

## 常见坑与最佳实践

- ⚠️ **条件里用 `===` 而不是 `=`**。`if (x = 1)` 是赋值，永远为真还会篡改变量。
- ⚠️ **`switch` 每个 `case` 记得写 `break`**，除非你是刻意合并 case。
- ⚠️ **警惕假值**：`if (count)` 在 `count` 为 0 时不会进去。要判断"有没有传值"，用 `count === undefined` 更稳妥。
- ⚠️ **循环别写成死循环**：`while` 的条件、递归的参数，都必须朝着"终止"的方向变化。真卡住了按 `Ctrl + C`。
- ⚠️ **数组用 `for...of`，对象用 `for...in`**。用 `for...in` 遍历数组会拿到字符串下标、还可能带出额外属性。
- 📌 **分清 `console.log` 和 `return`**：打印不等于返回。要让结果被后续代码使用，必须 `return`。
- 📌 **`===` 判断、`const` 优先、先定义后使用**——这三条贯穿全书。
- 💡 **函数要"小而专"**：一个函数只做一件事，名字就是它做的事（`randomInt`、`judge`）。这样代码才好读、好测、好复用。
- 💡 **提前 `return` 让代码更扁平**：与其把主逻辑塞进层层 `if`，不如在开头把不合法的情况 `return` 掉。

## 本章小项目：命令行猜数字游戏

### 需求

写一个能在终端玩的猜数字游戏：

1. 程序随机生成一个 **1~100** 的整数（藏起来）。
2. 玩家在终端**反复输入**自己的猜测。
3. 每次猜完，程序提示 **"大了"** 或 **"小了"**。
4. 猜中时，恭喜玩家，并告诉他**一共猜了多少次**，然后结束。
5. 玩家输入不合法（不是 1~100 的整数）时给出提示，且这一次不计入次数。

这个项目会同时用到本章几乎所有知识：**条件**（判断大小）、**循环**（反复猜）、**函数**（拆分逻辑）。

### 新知识：怎么读用户从终端输入的内容

前面所有程序都是"自说自话"地 `console.log`，不接收输入。要读终端输入，需要 Node 内置的 **`readline`** 模块。我们用它的 Promise 版本 `node:readline/promises`，配合 `async / await`，写起来最接近"同步思维"。

- **`async / await`** 是处理"异步操作"的语法。读用户输入是**异步**的——程序要**停下来等你敲键盘**，不能干等着阻塞一切。`await` 的作用就是"在这里暂停，等输入到了再继续往下走"。
- 你现在只需照着模板用，**背后的事件循环、Promise 原理是第 8 章的重头戏**，这里不展开。

> 📌 一个约定：`await` 只能写在 `async` 函数里。所以我们把整个游戏逻辑放进一个 `async function main()`，最后调用 `main()` 启动。

### 设计思路

先把大问题拆成几个"小而专"的函数：

1. `randomInt(min, max)`——生成指定范围的随机整数。
2. `judge(guess, answer)`——比较猜测和答案，返回 `"high"` / `"low"` / `"correct"`。
3. `main()`——主流程：生成答案 → 循环读输入 → 校验 → 判断 → 提示或结束。

主循环用"无限循环 + 猜中时 `break`"的结构，正好对应"不知道要猜几次，但知道什么时候停"。

### 完整实现

```javascript
// 文件：code/ch02/guess-number.js
// 运行方式：node guess-number.js

import * as readline from "node:readline/promises";
import { stdin as input, stdout as output } from "node:process";

// 生成 [min, max] 闭区间内的随机整数。
function randomInt(min, max) {
  return Math.floor(Math.random() * (max - min + 1)) + min;
}

// 判断一次猜测的结果。
function judge(guess, answer) {
  if (guess > answer) {
    return "high";
  } else if (guess < answer) {
    return "low";
  } else {
    return "correct";
  }
}

async function main() {
  const answer = randomInt(1, 100);
  let count = 0; // 有效猜测次数

  const rl = readline.createInterface({ input, output });

  console.log("我想好了一个 1~100 的整数，来猜猜看吧！");
  output.write("请输入你的猜测：");

  // 逐行读取输入：每输入一行并回车，就拿到 line。
  for await (const line of rl) {
    const guess = Number(line.trim());

    // 输入校验：不是 1~100 的整数就重来，不计数。
    if (!Number.isInteger(guess) || guess < 1 || guess > 100) {
      console.log("请输入 1~100 之间的整数哦～");
      output.write("请输入你的猜测：");
      continue;
    }

    count++;

    const result = judge(guess, answer);
    if (result === "high") {
      console.log("大了！再小一点～");
    } else if (result === "low") {
      console.log("小了！再大一点～");
    } else {
      console.log(`🎉 猜中了！答案就是 ${answer}，你一共猜了 ${count} 次。`);
      break; // 猜中，跳出循环
    }

    output.write("请输入你的猜测：");
  }

  rl.close(); // 关闭接口，让程序退出
}

main();
```

运行它：

```bash
cd code/ch02
node guess-number.js
```

然后就能在终端一次次输入数字、玩起来了。

### 逐段讲解

1. **引入模块**

   ```javascript
   import * as readline from "node:readline/promises";
   import { stdin as input, stdout as output } from "node:process";
   ```

   第一行把 `readline/promises` 整个模块引进来，命名为 `readline`。第二行从 `process` 里取出**标准输入 `stdin`**（键盘）和**标准输出 `stdout`**（屏幕），并顺手改名为 `input`、`output`，读起来更直观。`node:` 前缀表示"这是 Node 内置模块"（模块系统第 9 章详解）。

2. **`randomInt`——生成随机整数**

   ```javascript
   return Math.floor(Math.random() * (max - min + 1)) + min;
   ```

   `Math.random()` 返回一个 `[0, 1)` 之间的小数（能取到 0，取不到 1）。对 1~100：`max - min + 1` 是 100，乘上去得到 `[0, 100)` 的小数，`Math.floor` 向下取整得到 `0~99` 的整数，再 `+ min`（即 +1）就得到 `1~100`。这是一个值得记住的固定套路。

3. **`judge`——判断结果**

   一个"小而专"的函数：只负责比较，返回一个字符串标签，不打印、不读输入。把"判断"和"提示"分开，逻辑更清晰，也更好测试。这里用的正是 2.1 的 `if / else if / else`。

4. **`main` 里的准备工作**

   ```javascript
   const answer = randomInt(1, 100);
   let count = 0;
   const rl = readline.createInterface({ input, output });
   ```

   `answer` 用 `const`（一局游戏里答案不变）；`count` 用 `let`（要不断 `++`）。`createInterface` 创建一个"读写终端"的接口 `rl`，告诉它从哪读（`input`）、往哪写（`output`）。

5. **主循环：`for await ... of`**

   ```javascript
   for await (const line of rl) {
     const guess = Number(line.trim());
     ...
   }
   ```

   这是本项目的核心。`for await (const line of rl)` 会**逐行**读取用户输入：每当你输入一行、按下回车，循环体就拿到那一行文本 `line`。`line.trim()` 去掉首尾空白，`Number(...)` 把字符串转成数字（键盘输入永远是字符串，第 1 章讲过类型转换）。`for await` 是 `for...of` 的"异步版"，专门用来遍历这种"一次来一点"的数据流，第 8 章会正式讲。

6. **输入校验**

   ```javascript
   if (!Number.isInteger(guess) || guess < 1 || guess > 100) {
     console.log("请输入 1~100 之间的整数哦～");
     output.write("请输入你的猜测：");
     continue;
   }
   ```

   `Number.isInteger(guess)` 判断是不是整数（输入 `"abc"` 会转成 `NaN`，不是整数）。加上范围判断，任何不合法输入都会提示并 **`continue`**——跳过本轮剩余逻辑（尤其是不执行 `count++`），回到循环顶部等下一次输入。这里 `continue` 用得恰到好处。

7. **计数与判断**

   ```javascript
   count++;
   const result = judge(guess, answer);
   if (result === "high") {
     console.log("大了！再小一点～");
   } else if (result === "low") {
     console.log("小了！再大一点～");
   } else {
     console.log(`🎉 猜中了！答案就是 ${answer}，你一共猜了 ${count} 次。`);
     break;
   }
   ```

   通过校验才算一次有效猜测，`count++`。然后调用 `judge` 拿到结果，用 `if / else if / else` 分三种情况提示。猜中时打印结果并 **`break`** 跳出循环——游戏结束。

8. **收尾**

   ```javascript
   rl.close();
   ```

   关闭接口，释放对输入的占用，程序才会正常退出。忘了它程序可能一直挂着不结束。

> 💡 **用管道测试交互程序**：交互程序也能自动化测试。用 `printf` 造一串输入喂给它：
>
> ```bash
> printf "50\n25\n12\n" | node guess-number.js
> ```
>
> `\n` 代表一次回车。这样就不用手动一个个敲了，很适合快速验证逻辑。

### 扩展练习

1. **限制次数**：改成最多只能猜 7 次，用完还没中就公布答案并结束（提示：在循环里判断 `count`，或用带条件的循环）。
2. **给点方向感**：猜中前，额外提示"你已经猜了 N 次"，或当猜测和答案差距很大时提示"差得还远呢"。
3. **玩了还想玩**：一局结束后问玩家"再来一局吗？(y/n)"，输入 `y` 就重开。想想该用哪种循环把整个游戏再套一层（`do...while` 很合适）。
4. **难度选择**：开局先让玩家选范围（1~100 / 1~1000），把范围做成变量传给 `randomInt`。
5. **重构**：把 `judge` 从返回字符串改成直接返回要打印的提示语，比较一下两种设计哪个更好，为什么。

## 小结与自测

**本章重点回顾：**

- **条件**：`if / else if / else` 命中第一个为真的分支即停；`switch` 适合和固定常量比对，记得写 `break`（合并 case 时才故意不写）。
- **三元运算符** `条件 ? A : B` 适合在两个值间二选一，别嵌套太深。
- **循环**：`for`（知道次数）、`while`（先判断）、`do...while`（先做一次）；`for...of` 取值（数组）、`for...in` 取键（对象）；`break` 跳出、`continue` 跳过本轮。
- **函数**：函数声明、函数表达式、箭头函数三种写法；箭头函数单表达式可省 `return`。
- **参数**：默认参数、剩余参数 `...args`（现代首选）、`arguments`（老式，了解）。
- **`return`** 交回值并结束函数；没有 `return` 返回 `undefined`；打印 ≠ 返回。
- **提升**：函数声明会提升，函数表达式/箭头函数不会——统一"先定义后用"。
- **作用域**：函数作用域、块级作用域（`let/const`）；`var` 会泄漏出块；内层能读外层（闭包伏笔）。
- **递归**：函数调用自己，必须有终止条件；能用循环时优先用循环。
