# 前言 · 如何使用本书

欢迎！如果你此刻从未写过一行 JavaScript，也从未听说过 TypeScript，那么这本书正是为你写的。我们的目标只有一个：**读完并动手做完这本书，你能成为一名真正合格、能独立开发的 JS/TS 工程师（master）。**

## 这本书讲什么

现代前端与 Node.js 后端开发，几乎都建立在两门语言之上：

- **JavaScript（JS）**：浏览器和 Node.js 的"母语"，灵活、动态、无处不在。
- **TypeScript（TS）**：给 JavaScript 加上"类型"的超集（superset）。它编译后就是 JavaScript，但能在你写代码时就发现大量错误，是中大型项目的事实标准。

学习路线是有讲究的：**先把 JavaScript 学扎实，再学 TypeScript。** 因为 TS 只是给 JS 加了一层类型系统，JS 的一切语法与运行机制在 TS 中原样成立。跳过 JS 直接学 TS，会把"语言问题"和"类型问题"混在一起，事倍功半。所以本书的顺序是：

1. **JavaScript 基础与进阶**（第 1–9 章）——把语言本身吃透。
2. **现代 JS 与运行时**（第 10–12 章）——DOM、Node.js、正则。
3. **TypeScript 基础到类型系统进阶**（第 13–20 章）——从类型注解到"类型体操"。
4. **工程化与后端**（第 21–25 章）——工具链、测试、REST API、数据库、部署。
5. **综合大项目**（第 26 章）——把全书知识拼成一个完整应用。

## 每一章长什么样

为了让学习节奏稳定，每章都遵循同一套结构：

1. **本章目标**：读完你会掌握什么，一目了然。
2. **概念讲解**：核心知识点，配**大量示例代码**，逐段拆解——不仅告诉你"怎么写"，更解释"为什么"和"背后发生了什么"。
3. **常见坑 / 最佳实践**：新手最容易踩的雷，以及老手的习惯做法。
4. **本章小项目**：一个完整的小项目，带你把本章知识**用起来**。包含需求分析、设计思路、完整实现、逐段讲解，以及可选的扩展练习。
5. **小结与自测**：几句话回顾重点，几道题检验掌握程度。

> 📌 **关于代码**：书中出现的每段较完整的代码，在 `code/chNN/` 目录下都有对应的可运行文件，正文会标注文件名。**请务必一边读一边把代码敲进去跑一遍**——编程是"手上功夫"，只看不练等于没学。

## 准备工作

你只需要两样东西，第 0 章会手把手带你装好：

- **Node.js**：用来运行 JavaScript / TypeScript 代码（不用浏览器也能跑 JS）。
- **一个代码编辑器**：推荐 [VS Code](https://code.visualstudio.com/)，免费、对 JS/TS 支持极好。

如果你想在本地阅读这本电子书本身（带搜索、目录、深色主题），可以安装 [mdBook](https://rust-lang.github.io/mdBook/)，然后在 `master_book/` 目录运行 `mdbook serve --open`。当然，直接看 Markdown 源文件也完全可以。

## 给你的三条学习建议

1. **动手，动手，还是动手。** 每个示例都亲手敲、亲手改。把示例里的数字改一改、故意写错一处看看报什么错——报错信息是最好的老师。
2. **不要跳章。** 后面的章节大量依赖前面的概念。哪怕某章你觉得"好像会了"，也请把小项目做完再走。
3. **允许自己慢。** "精通"不是读得快，而是理解得深。遇到不懂的地方，停下来，重读，做实验，直到真的想明白。

准备好了吗？我们从第 0 章，搭好开发环境开始。
