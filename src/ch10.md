# 第 10 章 · 浏览器、DOM 与事件（概览）

## 本章目标

前九章我们一直在终端里用 Node.js 跑 JavaScript。但 JavaScript 的老家其实是**浏览器（browser）**——它当初就是为了让网页"动起来"而发明的。这本书的重点是 Node 后端，所以本章是一次**概览（overview）**：目标不是把你培养成前端工程师，而是让你彻底搞懂"一段 JS 在浏览器里到底怎么工作、能操纵什么"。这份理解在你以后调试网页、写全栈项目、甚至读别人前端代码时都用得上。

学完本章，你将能够：

- 说清楚 **HTML、CSS、JavaScript** 三者的分工，以及 `<script>` 标签放哪里、`defer` / `async` 有什么区别。
- 认识 **BOM**：`window`、`document`、`console`、`alert`、`location` 这几个浏览器给你的全局对象。
- 理解 **DOM（Document Object Model，文档对象模型）**：页面是一棵"节点树"，用 `querySelector` 等查到元素，读写它的内容、属性、样式，创建和删除节点。
- 掌握**事件（event）**：用 `addEventListener` 监听、读事件对象、`preventDefault`、理解**冒泡（bubbling）与捕获（capturing）**、用**事件委托（event delegation）**优雅地处理一堆元素。
- 读取表单、用 `localStorage` 存点小数据、知道 `fetch` 是怎么回事。
- 亲手做一个**不依赖任何框架**的待办清单网页。

📌 **重要前提**：本章代码要**在浏览器里打开 HTML 文件运行**，不是在终端 `node app.js`。原因很简单——`document`、`window` 这些东西只有浏览器才有，Node 里根本没有。

## 浏览器里的 JS：HTML / CSS / JS 三者关系

打开任何一个网页，背后都是三种语言在协作，各管一摊：

- **HTML（HyperText Markup Language）**：负责**结构与内容**。它用一层层"标签"描述"页面上有什么"——一个标题、一段文字、一个按钮、一个列表。
- **CSS（Cascading Style Sheets，层叠样式表）**：负责**外观**。颜色、字体、间距、布局都归它管。
- **JavaScript**：负责**行为**。用户点了按钮怎么办、输入了文字怎么响应、数据怎么变化，都靠 JS。

一个最小的网页长这样：

```html
<!-- 文件：code/ch10/demo-dom.html（片段） -->
<!DOCTYPE html>
<html lang="zh-CN">
  <head>
    <meta charset="UTF-8" />
    <title>DOM 与事件小实验</title>
  </head>
  <body>
    <h1 id="title">Hello DOM</h1>
    <p class="note">这是一段说明文字。</p>
    <button id="btn">点我</button>
  </body>
</html>
```

逐段拆解：

- `<!DOCTYPE html>`：告诉浏览器"这是一个标准的 HTML5 文档"，必须写在第一行。
- `<html lang="zh-CN">`：整个文档的根标签，`lang` 声明语言（有利于搜索引擎和屏幕阅读器）。
- `<head>`：**看不见**的头部信息。`<meta charset="UTF-8">` 声明用 UTF-8 编码（不写中文可能乱码）；`<title>` 是浏览器标签页上显示的标题。
- `<body>`：**看得见**的页面主体，用户在屏幕上看到的一切都在这里。
- `<h1>`、`<p>`、`<button>`：分别是一级标题、段落、按钮。每个标签可以带**属性（attribute）**，比如 `id="title"`（给元素起个唯一名字）、`class="note"`（给元素分类，可多个元素共用）。

💡 你不需要精通 HTML/CSS 才能学本章。记住"标签描述结构、属性给元素贴标记（尤其是 `id` 和 `class`，JS 靠它们找元素）"就够了。

### `<script>` 标签：JS 怎么进入页面

浏览器不会自己去找你的 JS 文件，你得用 `<script>` 标签告诉它。有两种写法：

```html
<!-- 写法一：内联（inline），直接把代码写在标签里 -->
<script>
  console.log("你好，浏览器");
</script>

<!-- 写法二：外链（external），引用一个 .js 文件 -->
<script src="app.js"></script>
```

小项目里我们会用外链——把 HTML（结构）和 JS（行为）分开，是良好习惯。

### 放哪里？`defer` 和 `async` 是什么

这是个非常容易踩坑、又非常重要的点。浏览器**从上到下**解析 HTML，遇到普通的 `<script>` 会**停下来**：先下载脚本、执行完，再继续往下解析。这带来两个后果：

1. 如果脚本放在 `<head>` 里、又想操作 `<body>` 里的元素——此时 `<body>` 还没被解析，元素根本**还不存在**，你的代码会拿到 `null` 而报错。
2. 大脚本会**阻塞（block）**页面渲染，用户盯着白屏等。

历史上的土办法是"把 `<script>` 放在 `</body>` 前面"，等所有元素都解析完了再执行脚本。这能用，但现代更推荐用 `defer`：

```html
<!-- 放在 head 里也没问题，因为 defer 会让它等到 DOM 就绪 -->
<script src="app.js" defer></script>
```

三种加载方式的区别：

| 写法              | 何时下载       | 何时执行                          | 顺序           |
| ----------------- | -------------- | --------------------------------- | -------------- |
| `<script>`        | 遇到即下载     | **立刻**，阻塞解析                | 按出现顺序     |
| `<script defer>`  | 边解析边下载   | **HTML 全部解析完后**、按顺序执行 | 保持出现顺序   |
| `<script async>`  | 边解析边下载   | **下载完立刻**执行（可能打断解析）| 谁先下完谁先跑 |

📌 记住结论：**操作页面元素的脚本，用 `defer`**（或放在 `</body>` 前）。这样脚本执行时，页面元素保证都已存在。`async` 适合与页面无依赖的独立脚本（如统计代码），本书小项目统一用 `defer`。

⚠️ **陷阱**：`defer` / `async` 只对**外链**脚本（有 `src`）生效。写在标签里的内联脚本加了也没用。

## BOM 简介：浏览器给你的全局对象

在 Node 里，全局对象是 `global`（或 `globalThis`）。在浏览器里，这个"万物之源"叫 **`window`**。它代表**浏览器窗口本身**，你能拿到的一切浏览器能力，几乎都挂在它上面。这一套围绕 `window` 的对象，统称 **BOM（Browser Object Model，浏览器对象模型）**。

几个你最常打交道的成员：

```javascript
// 这些代码都在浏览器控制台或 <script> 里跑（不是 node）

window.alert("弹个框");      // 弹出一个警告框，会打断用户
window.console.log("日志");  // 打印到开发者工具的控制台

// window 上的东西可以省略 window. 前缀，因为 window 是全局对象
alert("和上面一样");
console.log("和上面一样");

console.log(location.href);  // 当前页面的完整网址（URL）
console.log(location.host);  // 主机名，如 "example.com"
```

逐个说明：

- **`window`**：浏览器窗口/标签页。所有全局变量其实都是 `window` 的属性——你在顶层写 `var x = 1`，`window.x` 就是它（`let`/`const` 不会挂上去）。正因为 `window` 是全局对象，`window.alert` 可以简写成 `alert`。
- **`document`**：代表**当前页面的内容**。它是 BOM 里最重要的一员，也是**通往 DOM 的大门**——下一节整节都在讲它。
- **`console`**：开发者工具里的控制台。`console.log` / `console.error` / `console.warn` 是你在浏览器里调试的主力，和 Node 里用法一样。按 <kbd>F12</kbd>（或右键"检查"）就能打开开发者工具看到输出。
- **`alert` / `confirm` / `prompt`**：三个内置弹窗。`alert` 只提示；`confirm` 返回 `true`/`false`（确定/取消）；`prompt` 让用户输入一行文字。它们会**冻结页面**直到用户点掉，所以实际产品里很少用，学习和快速调试时方便。
- **`location`**：当前网址信息。`location.href` 是完整 URL，给它赋值（`location.href = "..."`）能跳转页面；`location.reload()` 刷新。

💡 打开任意网页，按 F12 切到 Console，直接敲 `document.title`，回车——你会看到这个页面的标题。控制台就是你探索 BOM/DOM 的游乐场。

## DOM：把页面看成一棵树

现在进入本章的核心。

当浏览器读入 HTML，它不会把 HTML 当成一堆字符串，而是解析成一个**对象的树**，这棵树就叫 **DOM（Document Object Model，文档对象模型）**。树上的每个"节点（node）"对应 HTML 里的一样东西：一个标签是一个**元素节点（element）**，标签里的文字是一个**文本节点（text node）**。

拿这段 HTML 举例：

```html
<body>
  <h1 id="title">Hello DOM</h1>
  <ul id="fruits">
    <li>苹果</li>
    <li>香蕉</li>
  </ul>
</body>
```

它在内存里对应的 DOM 树（简化）：

```
document
└── html
    └── body
        ├── h1#title      → 文本节点 "Hello DOM"
        └── ul#fruits
            ├── li        → 文本节点 "苹果"
            └── li        → 文本节点 "香蕉"
```

📌 **关键心智模型**：JavaScript 通过 `document` 拿到这棵树，然后**读它、改它、往上加节点、从上面删节点**——你改的是内存里的 DOM 树，浏览器会立刻把变化重新画到屏幕上。这就是"JS 让网页动起来"的本质。

### 查询：找到你要操作的元素

改一个元素之前，得先"抓住"它。最常用三个方法：

```javascript
// 文件：code/ch10/demo-dom.html（片段）

// 按 id 查（返回单个元素，找不到返回 null）
const title = document.getElementById("title");

// 按 CSS 选择器查第一个匹配的（更通用，推荐）
const note = document.querySelector(".note");        // 类名用 .
const firstLi = document.querySelector("#fruits li"); // 后代选择器

// 按 CSS 选择器查所有匹配的，返回一个 NodeList（类数组）
const items = document.querySelectorAll("#fruits li");
console.log(items.length); // 2
```

逐个拆解：

- **`getElementById("title")`**：按 `id` 找，参数**不带 `#`**。`id` 在页面里应当唯一，所以返回**一个**元素或 `null`。
- **`querySelector(选择器)`**：用**CSS 选择器**语法找**第一个**匹配的元素。`#title` 按 id、`.note` 按 class、`li` 按标签名、`#fruits li` 表示"`#fruits` 里面的 `li`"。这是最灵活、最常用的一个。
- **`querySelectorAll(选择器)`**：找**所有**匹配的，返回一个 **`NodeList`**。它长得像数组（有 `length`、能 `for...of` 遍历、能用 `forEach`），但不是真数组，没有 `map`/`filter`。需要的话用 `Array.from(items)` 转成真数组。

⚠️ **陷阱**：`querySelector` 找不到时返回 `null`，你再 `.textContent` 就会报 `Cannot read properties of null`。这通常是因为脚本在元素存在之前就跑了——回去看你的 `<script>` 是不是没加 `defer`。

### 读写内容：`textContent` vs `innerHTML`（务必看安全提醒）

抓到元素后，最常做的是改它的内容。有两个属性：

```javascript
const title = document.getElementById("title");

// 读
console.log(title.textContent); // "Hello DOM"

// 写：textContent 把内容当"纯文本"
title.textContent = "内容被 JS 改了";

// 写：innerHTML 把内容当"HTML 片段"解析
title.innerHTML = "被改成 <em>斜体</em> 了";
```

区别：

- **`textContent`**：读写**纯文本**。你赋进去的字符串里就算有 `<em>`，也会被当成普通字符原样显示，不会变成标签。
- **`innerHTML`**：读写 **HTML 字符串**。浏览器会把它**当成 HTML 解析**，`<em>斜体</em>` 会真的渲染成斜体。

`innerHTML` 很方便，但藏着一个严重的安全问题：

⚠️ **XSS（Cross-Site Scripting，跨站脚本攻击）**：如果你把**用户输入的内容**直接塞进 `innerHTML`，攻击者就能输入一段带 `<img onerror=...>` 之类的字符串，让恶意代码在别人的浏览器里执行——盗取登录凭证、冒充用户操作等。看这个反面例子：

```javascript
// ❌ 危险！千万别这样处理用户输入
const userInput = '<img src="x" onerror="alert(\'你被攻击了\')">';
someElement.innerHTML = userInput; // onerror 里的代码真的会执行！
```

（补充一个易被误解的点：直接把 `<script>...</script>` 通过 `innerHTML` 插进去，里面的脚本**不会**自动执行；但像上面 `<img onerror>`、`<svg onload>` 这类"带事件处理属性的标签"照样会触发执行。所以结论不变——绝不要拿用户输入喂 `innerHTML`。）

📌 **安全铁律**：**展示用户输入的内容，一律用 `textContent`，不用 `innerHTML`。** `textContent` 会把一切当纯文本，攻击者的标签只会原样显示成字符，无法执行。本章小项目里显示待办文字，我们坚持用 `textContent`。只有当 HTML 片段完全由你自己控制、不含任何用户数据时，用 `innerHTML` 才安全。

### 读写属性、class、style

除了内容，元素的属性和样式也能读写：

```javascript
const note = document.querySelector(".note");

// —— 属性（attribute）——
note.setAttribute("data-role", "hint"); // 设置属性
console.log(note.getAttribute("data-role")); // "hint"
console.log(note.hasAttribute("hidden")); // false

// data-* 自定义属性有个快捷入口：dataset
note.dataset.role; // "hint"（把 data-role 读成 dataset.role）

// —— class（推荐用 classList，而不是拼字符串）——
note.classList.add("highlight");    // 加一个类
note.classList.remove("old");       // 去掉一个类
note.classList.toggle("active");    // 有就去掉、没有就加上
console.log(note.classList.contains("highlight")); // true

// —— 行内样式 style ——
note.style.color = "crimson";       // 对应 CSS 的 color
note.style.backgroundColor = "#eee"; // CSS 的 background-color → 驼峰写法
```

要点：

- **属性**用 `getAttribute` / `setAttribute` / `hasAttribute` / `removeAttribute` 存取。`data-*` 开头的**自定义属性**还能通过 `element.dataset` 用驼峰名读写（`data-user-id` ↔ `dataset.userId`）——小项目里我们就用 `data-id` 把每条待办的 id 存在 DOM 上。
- **class** 强烈建议用 **`classList`**：`add` / `remove` / `toggle` / `contains` 四个方法，语义清晰、不会误伤别的类。别去手动拼 `className` 字符串，容易把已有的类覆盖掉。
- **style** 对应 CSS 的**行内样式**。注意 CSS 里带连字符的属性名，在 JS 里要写成**驼峰（camelCase）**：`background-color` → `backgroundColor`、`font-size` → `fontSize`。

💡 大多数时候更好的做法是：在 CSS 里预先写好几个类（比如 `.done { text-decoration: line-through; }`），JS 只负责用 `classList.toggle("done")` 切换类，把"长什么样"留给 CSS。小项目正是这么做的。

### 创建、插入、删除节点

页面上本来没有的东西，可以用 JS 现造出来再塞进去：

```javascript
// 1. 创建一个新元素（此时它还"游离"在内存里，不在页面上）
const li = document.createElement("li");
li.textContent = "橙子";

// 2. 插入到某个父元素里
const list = document.getElementById("fruits");
list.appendChild(li);        // 追加到末尾
list.prepend(li);            // 插到最前面
list.insertBefore(li, list.firstChild); // 插到指定节点之前

// 3. 删除
li.remove();                 // 删掉自己（现代写法，最简单）
list.removeChild(li);        // 或让父元素删掉某个子节点
```

拆解这套"创建 → 插入"的流程：

- **`document.createElement("li")`**：造一个 `<li>` 元素节点。此刻它还没进 DOM 树，屏幕上看不到。
- 给它设置内容/属性（`textContent`、`classList` 等），就像装配一个零件。
- **`appendChild` / `prepend` / `insertBefore`**：把这个零件安到 DOM 树上，屏幕才会显示出来。`appendChild` 加到末尾最常用。
- **`remove()`**：现代浏览器里删节点最简单——直接让节点删掉自己。老代码里常见 `parent.removeChild(child)`，效果一样。

⚠️ **性能小提醒**：如果要一次性加很多节点，别在循环里每次都 `appendChild`（每次都可能触发浏览器重新计算布局）。可以先把它们攒进一个 `DocumentFragment` 再一次性插入，或者像小项目那样先拼好再统一 `render`。数据量小时不必纠结。

## 事件：让页面响应用户

页面能"动"，核心在于**响应用户操作**：点击、输入、提交、按键……这些动作在浏览器里都表现为**事件（event）**。你的代码只要"订阅"感兴趣的事件，浏览器就会在事件发生时回调你的函数。

### `addEventListener`：监听事件

```javascript
const btn = document.getElementById("btn");

btn.addEventListener("click", (event) => {
  console.log("按钮被点了！", event.type); // "click"
  event.currentTarget.textContent = "已点击";
});
```

拆解：

- **`addEventListener(事件名, 回调函数)`**：给元素挂一个监听器。第一个参数是**事件名字符串**（`"click"`、`"input"`、`"keydown"` 等，注意**不带 `on`**）；第二个是**事件处理函数（event handler）**，事件发生时被浏览器调用。
- 处理函数会收到一个**事件对象（event object）**，习惯叫它 `event` 或 `e`。它装着这次事件的所有信息。
- 一个元素上可以 `addEventListener` **多次**，每个监听器都会被调用，互不覆盖。这比老式的 `btn.onclick = fn`（只能有一个）灵活得多。

💡 需要取消监听时用 `removeEventListener(事件名, 同一个函数引用)`。注意必须传**同一个函数**，所以匿名箭头函数是没法移除的——要移除就得先把函数存进变量。

### 事件对象：这次到底发生了什么

事件对象上你最常用的几个：

```javascript
input.addEventListener("keydown", (event) => {
  console.log(event.type);          // 事件类型，如 "keydown"
  console.log(event.key);           // 按下的键，如 "Enter"、"a"
  console.log(event.target);        // 事件"最初"发生在哪个元素上
  console.log(event.currentTarget); // 当前监听器挂在哪个元素上
});
```

- **`event.type`**：事件类型字符串。
- **`event.target`**：事件的**源头元素**——用户实际点/按到的那个元素。
- **`event.currentTarget`**：**当前监听器所在的元素**。在事件委托里，`target` 和 `currentTarget` 常常不是同一个，这个区别是委托的关键，下面细讲。
- 不同事件有各自的专属属性：键盘事件有 `event.key`，鼠标事件有 `event.clientX/clientY` 等。

### 常见事件与 `preventDefault`

几个最常打交道的事件：

```javascript
// click：鼠标点击
button.addEventListener("click", (e) => { /* ... */ });

// input：输入框内容变化时（每敲一个字都触发）
searchBox.addEventListener("input", (e) => {
  console.log("当前值：", e.target.value);
});

// keydown：按下键盘
box.addEventListener("keydown", (e) => {
  if (e.key === "Enter") console.log("按了回车");
});

// submit：表单提交（挂在 <form> 上）
form.addEventListener("submit", (e) => {
  e.preventDefault(); // 阻止浏览器默认的"跳转/刷新页面"
  console.log("在这里处理表单，页面不会刷新");
});
```

重点解释 **`preventDefault`**：很多事件有浏览器内置的**默认行为**——`<form>` 提交会刷新/跳转页面、点 `<a>` 链接会导航、右键会弹菜单。当你想用 JS 自己接管、不要浏览器插手时，就调 **`event.preventDefault()`** 取消默认行为。表单提交时几乎总要调它，否则页面会刷新，你的 JS 逻辑还没跑完就被打断了。

⚠️ 别把 `preventDefault()`（阻止**默认行为**）和 `stopPropagation()`（阻止**冒泡传播**，下节讲）搞混，它俩管的是完全不同的事。

### 冒泡与捕获：事件是怎么传播的

这是理解事件委托的前提，务必搞清楚。

当你点击一个 `<li>` 里的 `<span>`，事件**并不只发生在那个 `<span>` 上**。浏览器让事件经历三个阶段：

1. **捕获阶段（capturing phase）**：从最外层 `document` **向下**传，一路经过 `html → body → ul → li → span`，直到目标元素。
2. **目标阶段（target phase）**：到达真正被点的元素（`span`）。
3. **冒泡阶段（bubbling phase）**：再从目标**向上**传，`span → li → ul → body → html → document`，像水里的气泡往上冒。

```
                 ┌─ 捕获 ↓ ─┐        ┌─ 冒泡 ↑ ─┐
   document  →   html → body → ul → li → span（目标）
                                          然后沿原路往上冒回 document
```

关键结论：**默认情况下，`addEventListener` 在冒泡阶段触发**。也就是说，你在外层 `ul` 上监听 `click`，点它里面任何一个 `li`/`span`，这个 click 都会**冒泡到 `ul`**，从而触发 `ul` 的监听器。此时：

- `event.target` = 你**实际点到**的元素（比如 `span`）；
- `event.currentTarget` = **监听器所在**的元素（`ul`）。

```javascript
ul.addEventListener("click", (e) => {
  // 点 ul 里的任意子元素都会跑到这里（冒泡上来的）
  console.log("target:", e.target.tagName);        // 可能是 "SPAN"
  console.log("currentTarget:", e.currentTarget.tagName); // "UL"
});
```

如果想在**捕获阶段**监听，给 `addEventListener` 传第三个参数 `{ capture: true }`（很少用）。想**阻止事件继续冒泡**，调 `event.stopPropagation()`。

### 事件委托：一个监听器搞定一堆元素

有了冒泡，就能用一个非常实用的技巧——**事件委托（event delegation）**。

设想一个待办列表有 100 个 `<li>`，每个都要能点击。笨办法是给 100 个 `<li>` 各挂一个监听器：既啰嗦，又有个致命问题——**新添加的 `<li>` 没有监听器**，因为它是后来才创建的。

委托的思路：**别在每个子元素上监听，而是在它们共同的父元素上监听一次**。反正子元素的 click 都会冒泡到父元素，我们在父元素这里用 `event.target` 判断"到底点的是哪个子元素、点的是不是删除按钮"，再决定怎么办。

```javascript
// 只在父级 ul 上监听一次
listEl.addEventListener("click", (event) => {
  // event.target 是实际被点的元素，用 closest 往上找到它所在的 <li>
  const li = event.target.closest("li[data-id]");
  if (!li) return; // 点到空白处，不是任何 li，忽略

  const id = Number(li.dataset.id);

  if (event.target.classList.contains("del-btn")) {
    removeTodo(id);  // 点到的是删除按钮
  } else {
    toggleTodo(id);  // 点到别处 → 切换完成
  }
});
```

拆解这段委托逻辑：

- 监听器只挂在 `listEl`（那个 `<ul>`）上，**一次**。以后无论新增多少 `<li>`，它们的点击都会冒泡到这里，**自动**被处理——不用给新节点补挂监听器。
- **`event.target.closest("li[data-id]")`**：`closest` 从被点元素开始，**沿 DOM 树往上**找第一个匹配选择器的祖先（包括自己）。不管你点到的是 `<li>` 本身、里面的文字 `<span>`、还是删除按钮，都能定位到所属的那个 `<li>`。
- 用 `li.dataset.id` 拿回我们当初存在 `data-id` 上的编号，就知道操作的是哪条数据。
- 再用 `event.target.classList.contains("del-btn")` 区分：点的是删除按钮就删，否则就切换完成状态。

📌 **事件委托三步口诀**：① 在父元素监听；② 用 `event.target` / `closest` 找出真正的目标；③ 据此分派处理。它省内存、天然支持动态添加的元素，是前端开发的常用模式。小项目的列表点击和筛选按钮都用了它。

## 表单读取与 localStorage

### 读取表单

网页收集用户输入靠**表单（form）**。读取输入框的值就是读它的 `value` 属性：

```javascript
const input = document.querySelector("#new-todo");
console.log(input.value); // 用户当前输入的字符串

// 复选框/单选框看 checked
const agree = document.querySelector("#agree");
console.log(agree.checked); // true / false
```

- 文本框、文本域、下拉框：读 **`.value`**（永远是字符串，需要数字记得 `Number(...)`）。
- 复选框、单选框：读 **`.checked`**（布尔值）。
- 表单整体提交监听 `<form>` 的 `submit` 事件，并 `preventDefault()` 阻止刷新。

### localStorage：在浏览器里存点小数据

刷新页面后，JS 变量全没了。想让数据"活过刷新"，最简单的办法是 **`localStorage`**——浏览器提供的一个**键值存储（key-value store）**，数据存在用户本地磁盘，关掉浏览器也还在。

```javascript
// 存：键和值都必须是字符串
localStorage.setItem("username", "小明");

// 取：不存在返回 null
console.log(localStorage.getItem("username")); // "小明"

// 删 / 清空
localStorage.removeItem("username");
localStorage.clear();
```

⚠️ **只能存字符串**。要存对象或数组，得先用 `JSON.stringify` 转成字符串存，取出来再 `JSON.parse` 转回来——这正是小项目保存待办列表的做法：

```javascript
// 存对象/数组：序列化
localStorage.setItem("todos", JSON.stringify(todos));

// 取出来：反序列化（可能失败，用 try/catch 兜底）
try {
  const raw = localStorage.getItem("todos");
  const todos = raw ? JSON.parse(raw) : [];
} catch {
  const todos = [];
}
```

📌 `localStorage` 适合存少量、不敏感的数据（如界面偏好、草稿）。**不要**存密码、令牌等敏感信息。容量约 5MB，也别当数据库用。

## 简介 fetch：浏览器怎么向服务器发请求

到目前为止我们的数据都在本地。真实网页常常要向**服务器**要数据。浏览器发网络请求的现代方式是 **`fetch`**：

```javascript
// fetch 返回一个 Promise（还记得第 8 章的异步吗？）
async function loadUser() {
  const res = await fetch("https://api.example.com/user/1");
  if (!res.ok) throw new Error("请求失败：" + res.status);
  const data = await res.json(); // 把响应体解析成 JSON
  console.log(data);
}
```

简单拆解（本章只需看懂，第 23 章后端时会深入）：

- **`fetch(url)`** 发起一个 HTTP 请求，返回一个 **Promise**——正好用上第 8 章学的 `async/await`。
- `await fetch(...)` 得到一个**响应对象（Response）**。`res.ok` 表示状态码是否 2xx；`res.status` 是状态码（如 200、404）。
- `await res.json()` 把响应体解析成 JS 对象（也可用 `res.text()` 取纯文本）。

💡 你会发现：无论浏览器端的 `fetch` 还是后面 Node 端处理请求，底层都是同一套 **HTTP** 协议、同一套 **Promise/async** 异步模型。前九章打下的语言功底，在两端都通用——这正是学 JavaScript 的性价比所在。

## 常见坑与最佳实践

- ⚠️ **脚本跑太早拿到 `null`**：操作元素的脚本一定要在 DOM 就绪后执行。用 `<script defer>`（或放 `</body>` 前）。报 `Cannot read properties of null` 时，先怀疑这个。
- ⚠️ **`innerHTML` + 用户输入 = XSS 漏洞**：展示用户内容一律用 `textContent`。只有完全由自己控制、不含用户数据的 HTML 片段才用 `innerHTML`。
- ⚠️ **忘了 `preventDefault()`**：表单 `submit`、链接 `click` 有默认行为会刷新/跳转页面，用 JS 接管时记得阻止。
- ⚠️ **`preventDefault` 与 `stopPropagation` 混淆**：前者阻止**默认行为**，后者阻止**冒泡传播**，别用错。
- 💡 **优先事件委托**：列表、动态生成的元素，用委托在父级监听一次，省内存又天然支持后加的元素。
- 💡 **`querySelectorAll` 返回的不是数组**：需要 `map`/`filter` 时先 `Array.from(...)`。
- 💡 **样式交给 CSS 类**：JS 只 `classList.toggle` 切换类，别在 JS 里堆一堆 `style.xxx`，更好维护。
- 💡 **元素只查一次、缓存起来**：把 `querySelector` 的结果存进变量复用，别在每次事件里反复查询。
- ⚠️ **`localStorage` 只存字符串**：对象/数组要 `JSON.stringify` / `JSON.parse`，且解析要用 `try/catch` 兜底（存的数据可能损坏或格式变了）。

## 本章小项目：纯 JS 交互小页面

我们来做一个**不依赖任何框架**的待办清单网页，把本章所有知识点串起来：输入框 + 按钮添加待办、列表显示、点击项切换"完成"（划线）、删除按钮、按状态筛选、用 `localStorage` 持久化、用**事件委托**处理列表点击。数据模型呼应第 3 章的 todo：`{ id, text, done }`。

产出两个文件：`code/ch10/index.html`（结构 + 样式）和 `code/ch10/app.js`（逻辑）。**用浏览器打开 `index.html` 运行，不是 node。**

### 页面结构（index.html）

关键部分（完整样式见仓库文件，这里聚焦结构）：

```html
<!-- 文件：code/ch10/index.html（片段） -->
<h1>我的待办清单</h1>

<div class="add-row">
  <input id="new-todo" type="text" placeholder="今天要做点什么？" autocomplete="off" />
  <button id="add-btn">添加</button>
</div>

<div class="filters">
  <button data-filter="all" class="active">全部</button>
  <button data-filter="active">未完成</button>
  <button data-filter="done">已完成</button>
  <span id="count"></span>
</div>

<ul id="todo-list"></ul>

<!-- defer：等 DOM 解析完再执行，且不阻塞渲染 -->
<script src="app.js" defer></script>
```

拆解：

- 一个输入框 `#new-todo` 和添加按钮 `#add-btn`。
- 三个筛选按钮，用 `data-filter` 属性标记各自的过滤模式（`all`/`active`/`done`），后面 JS 用 `dataset.filter` 读它——又是 `data-*` 的用法。
- 一个空的 `<ul id="todo-list">`，列表项由 JS 动态生成。
- 脚本用 `defer` 引入，保证执行时上面的元素都已存在。

### 逻辑（app.js）：状态与持久化

```javascript
// 文件：code/ch10/app.js（片段）
const STORAGE_KEY = "ch10.todos";

function loadTodos() {
  try {
    const raw = localStorage.getItem(STORAGE_KEY);
    return raw ? JSON.parse(raw) : [];
  } catch {
    return [];
  }
}

function saveTodos() {
  localStorage.setItem(STORAGE_KEY, JSON.stringify(todos));
}

let todos = loadTodos();
let filter = "all";
```

拆解：

- `todos` 是唯一的**数据来源（single source of truth）**：一个 `{ id, text, done }` 数组。页面只是它的"投影"。
- `loadTodos` 启动时从 `localStorage` 读回并 `JSON.parse`，用 `try/catch` 防止数据损坏时崩溃。
- `saveTodos` 每次改动后把数组 `JSON.stringify` 存回去。
- `filter` 记住当前筛选模式。

### 逻辑：渲染

```javascript
// 文件：code/ch10/app.js（片段）
const listEl = document.querySelector("#todo-list");
const countEl = document.querySelector("#count");
const filterBtns = document.querySelectorAll(".filters button[data-filter]");

function render() {
  const visible = todos.filter((t) => {
    if (filter === "active") return !t.done;
    if (filter === "done") return t.done;
    return true;
  });

  listEl.innerHTML = ""; // 清空（这里是固定字符串，安全）

  if (visible.length === 0) {
    const li = document.createElement("li");
    li.className = "empty";
    li.textContent = "这里空空如也～";
    listEl.appendChild(li);
  } else {
    for (const t of visible) {
      listEl.appendChild(createItem(t));
    }
  }

  const left = todos.filter((t) => !t.done).length;
  countEl.textContent = `还剩 ${left} 项`;

  for (const btn of filterBtns) {
    btn.classList.toggle("active", btn.dataset.filter === filter);
  }
}
```

拆解：

- **`render` 是核心思想**：数据一变就重新渲染。它先按 `filter` 过滤出该显示的 `visible`，清空列表，再逐个生成 `<li>` 塞进去。这种"改数据 → 重画"的模式，正是后面 React/Vue 等框架思路的雏形。
- 清空用了 `listEl.innerHTML = ""`——这里赋的是**固定空字符串**，不涉及用户数据，安全。
- 计数直接从 `todos` 数过未完成的，用 `textContent` 写进 `#count`。
- 用 `classList.toggle("active", 条件)` 高亮当前筛选按钮：第二个参数为 `true` 就加类、`false` 就去类。

### 逻辑：创建列表项（坚持 textContent 防 XSS）

```javascript
// 文件：code/ch10/app.js（片段）
function createItem(todo) {
  const li = document.createElement("li");
  li.dataset.id = String(todo.id); // 把 id 存进 data-id，委托时靠它找回数据
  if (todo.done) li.classList.add("done");

  const span = document.createElement("span");
  span.className = "text";
  span.textContent = todo.text; // ✅ 用 textContent，绝不用 innerHTML

  const del = document.createElement("button");
  del.className = "del-btn";
  del.textContent = "删除";

  li.appendChild(span);
  li.appendChild(del);
  return li;
}
```

拆解：

- 逐个 `createElement` 造节点、设内容、拼装，再返回。
- **`li.dataset.id = String(todo.id)`**：把这条待办的 id 记在 DOM 的 `data-id` 上，事件委托时就能反查出点的是哪条。
- **`span.textContent = todo.text`**：待办文字来自用户输入，这里**必须用 `textContent`**。若用 `innerHTML`，用户输入 `<img src=x onerror=...>` 就会触发 XSS。这是本项目最重要的安全细节。
- 完成状态通过给 `<li>` 加 `done` 类实现划线（CSS 里 `li.done .text { text-decoration: line-through }`），样式交给 CSS。

### 逻辑：业务操作与事件绑定

```javascript
// 文件：code/ch10/app.js（片段）
function addTodo(text) {
  const trimmed = text.trim();
  if (!trimmed) return; // 空白不加
  todos.push({ id: Date.now(), text: trimmed, done: false });
  saveTodos();
  render();
}

function toggleTodo(id) {
  const t = todos.find((t) => t.id === id);
  if (t) { t.done = !t.done; saveTodos(); render(); }
}

function removeTodo(id) {
  todos = todos.filter((t) => t.id !== id);
  saveTodos();
  render();
}

// 添加：点按钮 / 按回车
addBtn.addEventListener("click", () => {
  addTodo(input.value);
  input.value = "";
  input.focus();
});
input.addEventListener("keydown", (event) => {
  if (event.key === "Enter") {
    addTodo(input.value);
    input.value = "";
  }
});

// 列表点击：事件委托
listEl.addEventListener("click", (event) => {
  const li = event.target.closest("li[data-id]");
  if (!li) return;
  const id = Number(li.dataset.id);
  if (event.target.classList.contains("del-btn")) {
    removeTodo(id);
  } else {
    toggleTodo(id);
  }
});

// 筛选：也用委托
document.querySelector(".filters").addEventListener("click", (event) => {
  const btn = event.target.closest("button[data-filter]");
  if (!btn) return;
  filter = btn.dataset.filter;
  render();
});

render(); // 首次渲染
```

拆解：

- 三个业务函数 `addTodo` / `toggleTodo` / `removeTodo` 都遵循同一节奏：**改 `todos` 数组 → `saveTodos()` 存盘 → `render()` 重画**。数据是主角，DOM 只是结果。
- `addTodo` 用 `text.trim()` 挡掉纯空白；用 `Date.now()`（毫秒时间戳）当简单的唯一 id。
- 添加支持两种触发：点按钮的 `click`、输入框里的 `keydown` 判断 `event.key === "Enter"`。
- **列表点击用事件委托**：只在 `#todo-list` 上监听一次，`closest("li[data-id]")` 定位到所属 `<li>`，再用 `classList.contains("del-btn")` 区分是删除还是切换。新增的待办无需重新绑定，天然生效。
- 筛选按钮同样用委托，绑在 `.filters` 容器上，读 `dataset.filter` 更新 `filter` 后重画。
- 最后 `render()` 一次，把从 `localStorage` 读回的数据画到页面上。

## 小结与自测

本章我们从 Node 回到浏览器，快速走了一遍前端最核心的三块：

- **三者分工**：HTML 管结构、CSS 管样式、JS 管行为。`<script>` 用 **`defer`** 引入外链脚本，保证 DOM 就绪后执行、不阻塞渲染。
- **BOM**：`window` 是浏览器全局对象，`document`、`console`、`alert`、`location` 都挂在它上面。
- **DOM**：页面是一棵**节点树**。用 `getElementById` / `querySelector` / `querySelectorAll` **查**；用 `textContent`（安全）/ `innerHTML`（有 **XSS** 风险）读写内容；用 `setAttribute` / `dataset` / `classList` / `style` 读写属性与样式；用 `createElement` + `appendChild` / `remove` **增删**节点。
- **事件**：`addEventListener` 监听，处理函数拿到**事件对象**（`type`/`target`/`currentTarget`/`key`）；`preventDefault` 阻止默认行为；事件经历**捕获→目标→冒泡**，默认在**冒泡**阶段触发；据此用**事件委托**在父级监听一次搞定一堆（含动态）子元素。
- **表单**读 `.value` / `.checked`；**localStorage** 存字符串（对象要 `JSON.stringify`/`parse`）做持久化；**fetch** 用 Promise 向服务器发请求。
